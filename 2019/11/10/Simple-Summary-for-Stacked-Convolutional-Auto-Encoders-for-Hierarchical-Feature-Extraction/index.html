Simple Summary for Stacked Convolutional Auto-Encoders for Hierarchical Feature Extraction

Posted on 2019-11-10

《Stacked Convolutional Auto-Encoders for Hierarchical Feature Extraction》

PS: Anto-Encoders 相关

Introduction:

  1. Abstract: 本文提出了一个 convolutional auto-encoder(CAE) 去进行无监督的feature learning, CAE的堆叠形成了一个CNN网络. 每个CAE都是用传统的不添加额外的regularization terms 的 on-line gradient descent进行训练. 认为Max-pooling layer对于学习 biologically plausible(生物上可解释性???) feature是至关重要的. 之后本文使用由CAE堆叠的CNN去得到了一个很好的性能(在MNIST和CIFAR10上)
  2. 认为unsupervised learning methods的主要目的是为了从unlabelled data中提取通用的有用的feature, 然后用来去进行检测并移除冗余仅仅保留最关键的鲁棒的有辨别力的特征部分.
  3. 提到了大多数方法都是基于encoder-decoder结构的 -> input会先转变为相对来说维度较低的特征空间(encoder), 然后将其expand去重新产生initial data(decoder). 一个layer被训练之后, 其code会被fed到下一个阶段去进行更高非线性依赖的构建.
  4. 提到了Auto-Encoder的思想, 这里略过. 提到了Denoising Auto-Encoder: 强调不添加任何的约束的时候, 传统的auto-encoders总会学习到indentity mapping. 一个规避方法就是使用probabilistic RBM方法, sparse coding 或者 denoising auto-encoders(DAs). DAs尝试着去重建noisy inputs. 会通过想原始clean input添加根据input image的特征得到的基于一个noise distribution中的随机变量 -> corrupted input.
  5. PS: Denoising AutoEncoder简单总结 具体总结见笔记

Method:

  1. Convolutional Auto-Encoder (CAE)
    • (1). 强调的一点就是 AEs 与 DAEs 都忽略了2D image的结构问题, 使用FC. 不仅仅对于图像的分辨率来说是不太可行的, 同时引入了参数的冗余, 使得每个feature都是全局的. 这与目前计算机视觉领域的趋势–利用局部特征不太符合.
    • (2). 本文使用的CAEs则就是之前提到的 -> 利用局部的特征. 整个处理的过程的数学公式都是类似的, 只不过是用卷积和激活函数代替一下. 同时为了减少参数自由度去使filter得到的是图片整体内一个相同类型的feature, 所以使用的bias是公共的, 而不是每个pixel一个. 激活函数选择的是 tanh. 使用 MSE 作为 Loss Function.
  2. Max-Pooling: 这里说max-pooling layer的作用通常是用来获取具有位移不变形的representation. 认为其有助于提高filter的选择性. 本文选择使用max-pooling的原因是希望feature detector能够变得更加的广泛被应用, 避免那些没有鲁棒性的琐碎的模式被学习(比如identity). 在reconstruction阶段, 系数的latent code 将会减少每个pixel进行decoding的filter的平均数量, 使得filter更加general.
  3. Stacked Convolutional Auto-Encoders (CAES): 将一些AEs进行堆叠得到一个深层次的结构就是CAES了.

问题:

  1. 利用本文的Max-Pooling部分的描述会不会有什么新的思路去当做有效信息进行提取.(比如unpooling…)
