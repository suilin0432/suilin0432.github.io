Simple Summary for Residual Bi-Fusion Feature Pyramid Network for Accurate Single-shot Object Detection

Posted on 2019-12-21

论文主题内核总结

单独列出的有名的pipeline

  1. RCNN系列
  2. SSD
  3. yolo
  4. point-based Anchor-Free (DenseBox, CornerNet)

PS: 找到了为啥大家都用sin和cos那个做embedding, Transformer文章中提出来的embedding方法, 目的是为了捕捉单词之间的相对位置(而且就是positional Embedding)(在CV中就也是为了捕捉相对位置, 那个log的变换之后可以看一下是不是这个目的) 其考虑的所谓的相对关系认为是 因为sin和cos性质 k+p处的位置向量是可以由位置k处得到 (个人认为同时还要引入非线性性质)

一、Encoder-Decoder结构:

  1. 《Stacked Hourglass Networks for Human Pose Estimation》(ECCV2016): 关于Encoder-Decoder在计算机视觉中的应用, 强调了浅层feature map的局部信息以及较为准确的位置信息(个人觉得是没有使用classification backbone所以位置信息准确)以及深层feature map的feature之间的关联信息是都是对最后的point estimation有很大益处的.
    • 优点:
      • (1). 通过加深网络的深度, 利用深层次信息之间的关联去提高一定的性能(利用了全局和局部的信息)
      • (2). 引入Encoder-Decoder结构能够起到去除噪声的作用去一定程度上解决模糊、光照等问题
      • (3). top-down以及bottom-up过程中计算量比较平衡是对称的模块
      • (4). 强调了特征融合带来的context的利用是有好处的
    • 缺点:
      • (1). 堆叠大量模块肯定会导致计算量、模型参数的增加
      • (2). 模型结构复杂
  2. 《DSSD : Deconvolutional Single Shot Detector》:见后面总结

二、feature mimic:

PS: (前两个针对克服小物体和大物体representation之间障碍)

  1. 《FEATURE INTERTWINER FOR OBJECT DETECTION》:同类别物体之间应该具有 More Compact Representation, 使用Teacher-Student Mimic做法. 先将数据集划分为两个集合, 一个为reliable set, 一个为less reliable set. 我们的目的是使用reliable set学习到的特征指导less reliable set的学习, 使得同类别的feature space更加的Compact, 从而扩大intra-class variance 减少 inter-class variance
    • 优点:
      • (1). 能够利用减小inter-class的差距, 增加intra-class的差距(使得同类别的信息表达更加紧凑, 也就是通过减小小物体和大物体的表达特征的差异)去达到用reliable set指导less reliable set的能力
      • (2). 模块方法可插拔
      • (3). 利用buffer去进行过往信息的存储, 防止在一个mini-batch中不会有同一类别在两个集合中出现的情况出现
      • (4). 使用OT(optimal transport)算法去进行更好的特征图的获得
    • 缺点:
      • (1). 比较依赖于可靠信息和不可靠信息的分类划分
  2. 《Perceptual Generative Adversarial Networks for Small Object Detection》: 希望消除的也是小物体和大物体的representation之间的gap, 希望能通过GAN将小物体的feature representation转换为大物体的representation, 但是认为直接学习representation的变换不是很容易, 选择根据低层次的feature map去学习小物体到大物体feature的residual细粒度的信息. 同时扩增了discriminator功能, 在本身的辨别生成的物体是大物体还是小物体的基础上还兼具这做detection loss的部分. (但是存在的疑问就是大物体的representation是怎么收集到的???… 文章也没有提及)
    • 优点:
      • (1). 使用GAN网络辅助进行目标检测训练学习比较新颖
      • (2). 通过减小小物体和大物体的语义gap以提高检测小物体的性能
    • 缺点
  3. 《Deformable ConvNet v2: More Deformable, Better Results》: 本文是基于deformable conv的基础上进行的改进, 其主要思想有3个: (1). 添加更多的deformable conv模块(使得感受野变化的范围更大) (2). 进一步增加deformable conv的可变化性(进一步的添加使用一个amplitude参数去进行影响程度的调节) (3). 通过与RCNN进行feature mimic去学习不关注RoI范围之外的, 避免受到非相关信息所产生的影响

三、训练策略(这里的训练策略有一点的广泛, 还包含了包括针对小物体的一些tricks):

  1. 《An Analysis of Scale Invariance in Object Detection – SNIP》: 本文是关于训练策略的, 强调应该在指定尺度上训练相应尺度的物体. 与finding tiny face文章思想相似, 认为预训练的ImageNet数据集中物体尺度都是相对来说比较大的, 而COCO数据集中的物体尺度都是很小的, 这存在着域迁移的问题. 本文首先对Image classification进行了一组实验, 发现了小物体确实在保证训练和测试时使用原图分辨率大小相同的时候进行放大能够提升正确的分类的概率, 但是也发现, 使用不同的分辨率的原图进行训练的时候对性能的影响是更大的. 并且针对detection问题使用了不同大小的training input image, 专用的针对小物体训练的网络, 多尺度随机放缩的detector进行实验, 发现对小物体放大是会带来益处, 但是当物体是极端大的时候就会带来性能的损失. -> 所以选择要在合适的scale上训练对应的物体 -> 只对落在该scale层上的物体进行反向传播去进行网络的学习, 其他的会被忽略掉. 同时最后为了解决high resolution image带来的GPU组员消耗问题, 会crop image到适合GPU训练的尺度上, 然后在crop出来的chip中选择包含最大数量小物体的chip进行训练, 知道所有的小物体都被包含为止(就是针对training过程)
  2. 《SNIPER: Efficient Multi-Scale Training》: 基于SNIP的一个改进, SNIP是使用image pyramid进行训练的, 可以发现, 当对image进行放大之后, 会有很多的区域是要被使用训练的, 但是在high resolution上, 很多区域实际上是没有用处的, 所以希望可以只利用有用的区域去进行训练, 从而减少处理的区域, 减少计算量. 本文做法是在SNIP的基础上再每个scale上截取指定大小的chips(按照一定的规律), 然后贪心的选择去覆盖valid GT box最多的那些chip去用来进行训练, 但是提到了这么做会导致假正率的上涨最后影响mAP, 所以会预先训练一个简单的找可能包含物体的RPN去进行invalid patch的学习, 即添加负样本进行训练. 本文最后可以在与SNIP performance相近的情况下只使用1/3的计算量. 因此作者认为传统的想法: “在high resolution的iamge上进行视觉任务是很有必要的” 是不一定正确的, 很可能是因为high resolution上图像的很多context是没有什么用的, 或者其感受野(ERF)是不足以覆盖全图的pixels的(因为一些论文实验表明感受野的增长并不是想之前认为的是指数上升的, 而是线性增长的)
  3. 《FSAF:Feature Selective Anchor-Free Module for Single-Shot Object Detection》: 具体内容见后面总结
  4. 《Seeing Small Faces from Robust Anchor’s Perspective》: 本文个人认为主要是对网络设计的a bag of tricks, 包含了四个点, 但是针对的主要问题就是相对于大脸来说小脸是不容易被初始的anchor所cover的(overlap很小), 作者做了一个实验表明了这个现象的存在, 然后本文主要贡献点就是: 提出了一个EMO Score去衡量一个尺度的face是否容易被anchor所overlap(本文为了简化计算进行了face中心点是均匀分布的以及设定face scale与anchor scale大小一样大的情况下的两个约束)得到了小脸期望IoU比较小的事实以及一个公式, 这个公式表明了小脸的overlap的大小与anchor stride是严格相关的, 基于这个点进行了两个个改进 -> (1). 因为传统的anchor是基于feature map的中心设置的, 所以anchor stride = feature stride, 所以减小feature stride有利于性能的提升(因此使用双线性插值/双线性插值+residual/diconv进行感受野保持的情况下的feature map尺寸的提升) (2). 不进行feature map的改变, 而仅仅去改变anchor stride的大小, 也就是在一个feature map的grid上设置多个anchor. 额外的两个改进是: (3). Face Shift Jittering: 对图像进行shift -> 使得脸和anchor的overlap增加 (4). Hard Face Compensation: 当一个脸和所有的anchor的IoU阈值都小于规定的positive阈值的时候, 将几个IoU相对较高的 额外的anchor当做positive进行训练
  5. 《Finding Tiny Faces》: 本文强调的点主要有三方面: (1). 物体尺度 (2). 图片分辨率 (3). context对小物体检测的重要性. (1). 关于尺度问题和SNIP类似, 都认为进行训练的样本数量会导致针对某一尺度的物体的性能更加好, 强调了ImageNet进行预训练的物体大小普遍都是中尺度的, 所以尽管COCO的物体大小小尺度偏多, 但是仍然是对中尺度物体的检测效果最有利的, 所以就针对这一点进行 2, 0.5, * 1的三个尺度, 希望小物体和大物体都能在某一尺度上放缩到中物体的范围内. (2). 其次对分辨率问题, 强调的是特征图的分辨率, 通过实验说明了浅层(高resolution)的feature map对小物体的检测是很有效的. (3). 对于context对小物体的重要性, 同样是做了一组实验, 发现对大物体来说使用大的感受野与使用小的感受野效果差不多(文章认为及时感受野小于大脸也可以通过部分推断局部, 但是其他一些文章观点和这个恰恰相反), 而对于小物体来说使用大的感受野的效果要明显变好 -> 因此最后选择使用一个固定的相对来说大的感受野.

四、backbone:

  1. 《CBNet: A Novel Composite Backbone Network Architecture for Object Detection》: 见后面总结
  2. 《DetNet: A Backbone network for Object Detection》 (没看过)
  3. 《ScratchDet: Training Single-Shot Object Detectors from Scratch》 (没看过)

五、Anchor-Free做法:

(一)、Point-based:

  1. 《Objects as Points》: (PS: 这是真的CenterNet): 本文的思想是通用的, 可以应用到包括2D detection、3D detection、pose estimate等问题. 本质思想就是先预测物体的center, 然后去estimate各个key point的offset. 相比于CornerNet, ExtremeNet来说省去了同一物体点的匹配阶段, 从而减少了时间的消耗以及误匹配的问题. 除此之外本文的训练方法和思想基本都是借鉴的CornerNet. 本文使用backbone的时候都会对backbone进行改造成encoder-decoder结构, 个人认为是因为是estimate point的思路, 其对精读相对来说要求较高, 同时提高resolution能够有效避免一个point中出现两个objext的可能性而带来漏检, 所以使用了encoder-decoder的结构思想. 本文最后抛弃了NMS的使用, 带来了一定性能的提升, 同时可以是用resnet-18, DLA-34这种轻量级的backbone上带来相当快的速度, 使用hourglass-104等重backbone也相对来说不是非常慢.
  2. 《CenterNet: Keypoint Triplets for Object Detection》: (PS: 假的CenterNet) 本文和Objects as Points的思想完全不同, 本文是基于CornerNet的一个改进, 针对的问题是CornerNet针对边界的检测比较敏感, 但是在组合的时候使用feature embedding导致匹配出错的情况大量出现(即针对误检情况作出改进, 仍然使用原来的左上右下思路, 但是添加额外信息去进行进一步筛选), 所以作者提出了三点的检测, 在左上、右下角点的基础上希望中心点能够用来帮助确定这个bbox是否是一个正确的object. 为了解决这个问题文章提出了center-pooling 通过取水平和垂直方向上的最大点去希望获取到proposal的中心部分, 并额外的预测了一个center分支来进行proposal的检验, 在组成的bbox的中心区域去检测是否有这个类别的中心点, 依照此决定是否保留(置信度取三点的平均值). 同时考虑到central region的大小的定义应该针对不同大小的bbox而设置. 本文对corner pooling部分也进行了改进, 提出了cascade corner pooling, 希望不仅仅是通过物体的top、left、right、bottom most去定义角点, 同时希望能够看向物体的内部来定位角点, 所以先用中间信息去英文边界点, 然后在用边界点去确定角点. PS: 本文是training from scratch, 训练Loss什么的还是总体思路借鉴的CornerNet.
  3. 《Grid R-CNN》 && 《Grid R-CNN Plus: Faster and Better》: 基于CornerNet思想的改进的对RCNN pipeline的改变, 其基于多点去进行定位, 先要解决的CornerNet两点定位只是对物体边缘敏感以及仅仅两个点约束较少容易出错而带来的不稳定(因为通常来说物体角点在物体之外, 而不是在物体之内), 通过使用全卷积的网络结构代替了传统Faster RCNN最后的bbox回归的几个FC层去更加准确的predict bbox的各个参数, 并强调了点之间的位置关联性以及约束性, 以及不同point之前进行预测的heatmap去进行feature fusion进一步的加深去进一步的提取特征信息来更好的定位.
    • PS: Grid R-CNN Plus的改进点主要在于对速度上的改进, 因为在之前的表述中, 每个Grid point的预测都是基于全图大小的, 但是实际上, 点的所在区域有很大一部分区域是不可能存在的, 因为正样本的IoU要求是>0.5, 因此每个点的可能存在的空间都只是原来的四分之一就好(保证了点的分布是一个正态分布, 而不是像之前的那样除了可能出现的1/4的区域剩下的区域都是不可能出现的.)(其实可能出现的区域并不一定只有1/4 但是个人认为是因为其他区域出现的概率太小了, 不应该一个RoI中物体的大小只占很小的一块区域, 而大部分的信息是背景区域), 另外将两个NMS的第二个去掉了, 一定程度上说明了信息足够的确定准确.
  4. 《CornerNet: Detecting Objects as Paired Keypoints》: Anchor-Free浪潮的比较早的一篇文章, 总体思路就是想要摆脱预先设置的anchor, (1). 避免密集的anchor带来的positive-negative imbalance问题 (2). 避免需要设定的大量的超参数. (3). 认为角点仅仅依赖于两条边, 但是中心是依赖于4条边的, 所以相对来说信息利用更容易(不是很认可) -> 因此提出了使用左上和右下两个角点去进行一个物体的标定, 但是因为没有办法确定哪两个点是同一个物体的, 所以采取了对每个点进行feature embedding的学习然后依据距离去匹配两个点. 为了提高角点的检测准确率, 本文提出了corner pooling, top-left角点要用一个分支去提取改点右侧最大值的feature map, 一个分支提取改点下面最大值的feature map. bottom-right则是上面和左面最大值的两个branch. 针对角点Loss会在一个范围(保证最小的overlap大于阈值情况下)权重是衰减的, 同时还会为了保证feature embedding的正确性, 设计了pull和push loss, 促进成对的corner的feature embedding差异减小, 不成对的feature embedding差异变大.

(二)、类似anchor-based

  1. 《FSAF:Feature Selective Anchor-Free Module for Single-Shot Object Detection》: 具体内容见后面总结
  2. 《DenseBox》: 没有仔细看过, 但是是这方面的算是最早的文章了吧, 而且创新点也很多.

六、Feature Fusion Module:

PS: 本类文章比较多, 而且通常来说思想是比较简单的

  1. 《CBNet: A Novel Composite Backbone Network Architecture for Object Detection》: 见后面总结
  2. 《Accurate Single Stage Detector Using Recurrent Rolling Convolution》:主要思想就是浅层次和深层次的信息都相互的去进行fusion, 并且会像RNN一样分stage的继续fusion
  3. 《Receptive Field Block Net for Accurate and Fast Object Detection》:具体改进见后面
  4. 《Libra R-CNN: Towards Balanced Learning for Object Detection》:具体改进见后面, 这篇文章针对三个问题进行了改进
  5. 《DSSD : Deconvolutional Single Shot Detector》: 见后面总结
  6. 《CFENet: An Accurate and Efficient Single-Shot Object Detector for Autonomous Driving》: 本文针对的仍然是feature fusion的传统的两个方面: (1). 加深网络模型, 提高感受野 (2). 进行feature fusion 思想简单,但是有效. 比较有创新的地方就是在做网络加深或者feature fusion后的信息迁移处理时的模块中用了 1 k k 1以及组卷积这种减少网络计算量的结构去进行计算量的减少. 针对小物体来说确实还是有效的.
  7. 《Weaving Multi-scale Context for Single Shot Detector》: 本文的出发点是feature fusion模型计算量太大了, 消除了SSD带来的速度上的优势, 希望能够通过light-weight的模块进行迭代的操作去达到一个好的效果. 因此本文选择不断的将每个scale的相邻的两个模块的feature map进行fusion, 然后到下一个阶段, 希望能够逐步的引入不同层次的信息, 认为不应该直接跨度过大的去进行feature的fusion. 同时后面提出了对该结构的加速简化模型的操作. (吐槽文章讲的一点都不清楚…)
  8. 《Enhancement of SSD by concatenating feature maps for object detection》: (RSSD) 本文是单纯的进行feature的fusion操作, 用两种操作列举出来了三种可行的处理方案进行分析 (1). 只使用pooling: 会将浅层的信息直接pooling之后和用conv得到的下一层进行concate (2). 只使用deconv: 会将深层次的feature map进行deconv到浅层上进行concate (3). 使用pooling和deconv: 两种操作都要 (个人认为前两个操作都因为太直接粗暴没有像其他作品一样在后面用卷积操作去处理进行一下语义信息的转变导致了效果的退化(不如SSD), 最后一个行为更像是完全依靠计算量的大量上升而达到的)
  9. 《Context-Aware Single-Shot Detector》: (CSSD, DiCSSD, DeCSSD) PS:这篇文章也强调了感受野的作用, 但是本质上是特征融合. 本文是基于感受野提出的特征融合, 强调了之前的研究普遍认为指数增长的理论感受野就是实际的感受野, 但是已经有论文实验表明经验感受野和理论感受野的开根号是成线性关系的, 也就是随着层数的增加、网络的加深, 性能的提升很可能是因为感受野达到了一定的有效范围而提升的(VGG的理论感受野达到了inputsize, 但是经验感受野是远远不够的), 同样和finding tiny face的观点一致->小物体需要大的感受野, 最浅层的那个高resolution的feature map是最适合小物体检测的, 因此希望能够将不同感受野的feature map进行fusion去得到feature map. 文章提出了两种做法: (1). 使用deconv: 将深层次的feature map使用deconv放大到与最浅层feature map相当的尺度, 然后进行fusion之后输出作为detection的feature map其他尺度则保持原来的输出. (2). 使用diconv: 通过空洞卷积进行处理, 在每个尺度上得到感受野不同的feautre map, 然后将这些feature map进行融合.
  10. 《FSSD: Feature Fusion Single Shot Multibox Detector》: 不像传统的做法将深层次的逐步的与浅层次的feature map进行fusion, 而是选择一起fusion到一个大的feature map之后, 利用这个feature map去进行下面的一个个深层的低resolution feature map的生成. 去进行预测
  11. 《M2Det: A Single-Shot Object Detector based on Multi-Level Feature Pyramid》: AAAI2019的文章, 最新的文章. 本文结构上就是疯狂的进行模块的堆叠, 看上去不是很漂亮, 但是有着新的观点, (1). 强调原来的backbone是用来classification的, 本文其实没有对这个做出太多的强调, 只是将backbone中的两层进行了特征融合操作作为base feature map (2). 强调的是传统的feature pyramid只是简单的在不同的resolution上进行feature map的fusion, 但是认为只是简单的根据物体的size去做的解释, 没有考虑到实际物体的语义信息是否符合. 所以希望不仅仅可以利用多尺度的feature map, 同一尺度的feature map需要融合不同深度(语义抽象程度)的feature信息(即希望物体可以在适当尺度、适当语义信息的feature map上被发现). 因此将设计了TUM结构(encode-decoder结构)用来提取当前深度的语义信息, 将得到的feature pyramid提取出来, 然后取出来high resolution的feature map与base feature进行fusion之后继续进行TUM结构的堆叠, 拿到下一个feature pyramid, 这样堆叠多次之后得到了好几组feature pyramid, 将这些feature pyramid的相同层次通过SFAM module得到最后的feature map(会使用SE module, 因为显然按照想法, 只有某一深度的pyramid中的feature map才包含这个物体的准确信息, 所以应该用SE module去进行channel-wise的module attention).

八、特征对准问题:

  1. 《Cascade RPN: Delving into High-Quality Region Proposal Network with Adaptive Convolution》: 本文说是根据RPN角度进行出发, Cascaded去进行anchor box的refine. 但是本质上是解决feature的对齐问题, 其和AlignDet的思想几乎可以说是完全一致, 只是讲故事的时候出发的角度不太一样
  2. 《Revisiting Feature Alignment for One-stage Object Detection》: AlignDet, 文章的思路就是, one-stage方法相较于two-stage方法差距就是在于two-stage在RPN筛选可能roi的时候会进行一次anchor regression, 这个操作使得所利用的feature与判断物体类别的feature是基本对齐了的, 同时RoIPooling的对准操作有利于位置敏感性的建立, 但是在one-stage上最后的anchor的估计是一步到位的, 一个点所利用的区域就是上一个特征图的卷积核对应位置的特征, 当物体并不在这个区域的时候, 就会存在这特征不对齐的问题. 认为传统的方法: (1). multi-stage: 部分解决了因为scale导致的misalignment, 但是并没有解决aspect ratio所导致的misalignment. (2). refineDet, 虽然借鉴了two-stage的思想, 进行一个两阶段的anchor的refinement, 但是其利用的仍然是不对准的信息. 文章通过了一个名叫RoIConv的东西去进行特征的对准, 实际上就是首先利用一个隐式的anchor去做简单的anchor参数的predict与refine, 之后去用RoIConv进行一个conv采样点的偏移对齐, 然后用这个采样点达到对齐的效果. -> 利用这个结果去进行分类以及进一步的refinement.
  3. 《Deformable Convolutional Networks》: 本文在backbone中提到了pool存在着特征不对准问题, 对位置敏感的任务(object detection 尤其是 segmentation)有着很大的影响, 进行了residual模块的一定改动使得backbone变得特征对准.

九、通过在模块中提高精度、考虑位置敏感信息、不同尺度物体的信息的思想:

  1. 《Feature Selective Networks for Object Detection》: 认为除了原本Faster-RCNN提取的CNN特征之外, 还应该引入sub-region与aspect ratio信息去进行提取了的特征信息的改善(也可以看做是利用Attention机制)(因为本着的思想就是在物体的不同子区域中以及不同的aspect ratio时, 特征的表现应该是有差异的.)
  2. 《R-FCN: Object Detetion via Region-based Fully Convolution Networks》: 本文针对的点就是detection使用的pre-train backbone是基于classification训练的, 其是对位置不敏感的, 但是detection任务是对位置敏感的, 认为Faster-RCNN为了达到位置敏感, 强行加入了RoI-pooling引入了位置信息, 但因此需要针对每个RoI进行feature的后续处理, 严重的影响了速度, 本文提出了一个全卷积的网络, 其将产生具有位置敏感性的score-map, 利用这个位置敏感性的score-map直接达到与RPN weight-share的目的, 既减少了计算量, 同时又保证了准确性. 本文是通过将一个物体的特征划分为 $k^2$ 个模块达到的(本文是区3*3即左上,上,右上等9个位置), 然后每个类别都提取一个关于每个位置特征的信息, 最后利用RPN层预测出来的RoI进行Position-sensitive RoI pooling, 做法是取grid覆盖范围内的所有position-sensitive score map对应pixel的均值, 然后会对所有的类别都进行取样, 得到一个(类别+1)(因为bg所以+1)channel的最终feature map, 然后使用voting去得到最后的结果
  3. PS: 其实Grid R-CNN也可以放在这里进行考虑
  4. PS: 其他包含这个思想的, 使用encoder-decoder结构的, Mask-RCNN

十一、Loss改进

  1. 《Libra R-CNN: Towards Balanced Learning for Object Detection》: 具体改进见后面总结
  2. 《Bounding Box Regression with Uncertainty for Accurate Object Detection》:(本文并没有看仔细做法, 只是搜论文的时候看了一下摘要和Introduction) 文章认为数据集通常会尽可能的使GT BBox的定义更加清晰, 到那时大多数情况下的bbox的label仍然是模糊不清的((1). 标注不准确(COCO中存在很多这样的现象) (2). 遮挡带来的模糊, 认为这个会导致label定义的不清楚或者并不是最优的label定义, 导致了网络对物体边界的学习比较模糊(PS: 但是对CornerNet都认为对物体边界关注的很多, 会不会也存在着这个问题的影响)) 所以本文认为传统的L1 Loss并没有考虑到这种模糊的情况, 导致了学习得到的最高的cofident的anchor并不是最优的, 作者选择将bbox prediction分布和GT bbox分布分别看为高斯分布和狄拉克函数, 然后loss使用两者的KL divergence作为新的bbox regression loss. 同时在NMS的处理中也引入了NMS.
  3. 《Focal Loss for Dense Object Detection》: 强调了positive-negative样本不均衡问题, 想要进行hard mining, 通过降低easy negative的权重达到效果, 同时本文也提出了retinaNet, 只是简单的采用了FPN的融合方式同时在detection的时候在classification与regression两个分支上多接了几个卷积层. (PS: 之前比赛表明对密集的物体来说retinaNet + focal loss效果好像特别好…)

十二、Cascade思想

  1. 《Single-Shot Refinement Neural Network for Object Detection》: (这个也可以看作是one-stage和two-stage结合的思想.) 首先本文的目的就很明确: 想要结合two-stage的精准以及one-stage快速的两个优点, 本文观点是one-stage性能不好是因为类别不平衡问题(个人认为主要是指的过多的negative anchor导致的类别不均衡问题), 提到了two-stage的优点之一是, (1). 首先使用类别无关的sampling去处理类别不平衡问题, (2). 其次对bbox regression能够使用两阶段的cascaded处理方式进行回归, (3). 最后使用two-stage的feature去进行物体的描述(首先将物体区进行二分类分为前景以及后景, 之后将物体进行根据feautre进行类别的进一步判定以及bbox进一步regression), 所以为了达到这个效果, 希望能够在one-stage模型中进行two-stage的二次筛选以及二次回归操作, 其首先在SSD forward过程中进行一次anchor refinement并进行anchor的筛选, 然后将深层次的信息逐步与浅层的信息进行融合, 利用得到的融合信息和筛选过的refine一次的anchor进行最后的分类以及回归. 这样就达到了two-stage两次回归的优势, 以及通过第一次的初步筛选达到粗分类的效果, 同时减少class imbalance问题的存在.
  2. 《Cascade R-CNN Delving into High Quality Object Detection》:见后面总结
  3. 《CASCADE MASK GENERATION FRAMEWORK FOR FAST SMALL OBJECT DETECTION》: 本文主要思想就是, feature pyramid并不是万能的, image pyramid很多时候还是很有必要的, 尤其针对小物体, 因为小物体在low resolution图片上的信息通常只有几个或者几十个pixel, 经过几次卷积过后信息就已经消失了, 所以使用image pyramid有时候还是比feature pyramid要好的(个人以及其他文章总结的2333), 但是强调了直接处理整张image的话太慢了, 所以希望能够在高层次上处理尽可能少的pixels(PS: 想法和SNIP很想, 但是做法不一样), 因此希望能够级联式的向下去进行处理, 在前一个阶段去进行mask map的预测(RoI-level的拒绝, stride-size大小的pixel mask(个人认为一是因为操作方便的原因, 二是因为pixel-level的拒绝太浪费时间))(与RPN分支共享feature map), 然后在下一个阶段去只处理没有被mask掉的区域. 注意的点是因为拒绝掉了很多区域, 所以整个图片的有效像素并不是连续的但是图片尺寸还是不变的, 所以在处理的时候如果简单的通过conv操作那么就并不会减少时间, 而是采用RoI conv的处理, 利用conv的实现即im2col进行计算. 最后在小物体数据集上可以取得不错的成绩.
  4. 《AutoFocus-Efficient Multi-Scale Inference》: 本文作者是SNIP系列的作者, 可以说文章的思路是顺着SNIP-SNIPER这条线的后续处理, 关注点这次不再是单纯的训练策略了, 而是inference与train过程都采取的pipeline. 本文关注点在于小物体的检测, 出发点思路是基于SNIP系列的使用image pyramid以及不同尺度feature map的不同instance训练分配策略, 同时继续延续SNIPER的不使用全图的所有的pixel进行训练以减少时间的思路. 本文在前一stage进行可能包含小物体的Focus Pixel的训练, 然后将Focus Pixel包裹成一个有着最小大小以上的Focus Patch, 将这个Focus Patch放大到训练尺寸去进行下一次的训练. 对小物体的性能提升很大, 甚至超过了深度的feature fusion的方法.
  5. 《Supervised transformer network for efficient face detection》: 本文是《CASCADE MASK GENERATION FRAMEWORK FOR FAST SMALL OBJECT DETECTION》思路的主要来源, 思路两者十分相像, 只不过后者将对mask map以及RPN预测的feautre map共用了, 且针对的领域不同而已.

十五、Attention

  1. 《An Empirical Study of Spatial Attention Mechanisms in Deep Networks》: 本文将在detection任务中近些年来提出的attention机制进行了一个总结, 然后尝试了使用多种组合去验证我们该去怎么做. 文章将这些attention机制分为了4类:
    • PS: 通常来说query和key指的都是image pixels, feature map pixels或者RoIs
    • (1). query and key content: 比如non-local(这里query是单个的pixel, key是整个feature map的pixels, 虽然你从non-local模块形式上来看两者是一个东西, 但是实际上non-local计算每个位置的时候都要考虑到整张图的效果, 其f(x_i, x_j)可以看做相似度)
    • (2). the query content and relative position: 比如deformable conv(可以看做为特殊的这个类别的attention)(其query是每个pixel的value去预测得到offset map, 后续的conv采样的时候进行的偏移是位置有关的), PSANet(这篇文章没有读过, 其是强调相互关注去得到权重的, 一个点的权重有其本身到其他点以及其他点到它的相对位置共同影响)
    • (3). the key content only: 比如GCNet(这篇文章将non-local每个位置都要考虑一个全局的attention改为考虑一个公共的全局的attention map, 这个时候就只有f(x_i), 而不是f(x_i, x_j)了)
    • (4). the relative position only: 按照文章分析定义来说, 普通的conv就是具有relative position only性质的attention.
    • 本文还将Transformer Attention的几种不同形式进行了划分, 分析了一下每种都对应着什么attention类别, 之后文章列举了几种将不同形式的attention组合的模块, 然后进行实验, 在object detection, segmentation, translation几个任务上进行实验去找哪些组合会有很好的效果, 最后得到的结论是: 在self-attention(key和query来自于同一个set, PS: CV领域中通常都是self-attention)中起到关键作用的是 query content以及relative position共同作用以及key content only. query-sensitive的模块通常来说起到的作用比较少甚至没有作用(PS: 这里指的都是已经引入了其他的attention之后再引入, 直接引入还是有作用的)但是反而会引来巨大的计算量损耗. 在encoder-decoder attention中, query&key content项起到的作用比较大, 甚至只需要这一项就可以了.
    • PS: 这可能也是一个解释GCNet有效的角度吧.
  2. 《Non-local Neural Networks》: 本文的出发点是: 目前的神经网络的操作都是local的(不仅仅是conv, RNN也是local的, 因为只考虑相邻的时间点), 希望能够提出一个通用模块去利用全局信息以捕捉长程的依赖关系. 首先对传统的conv的缺点进行了分析, 认为传统的conv是依靠多次操作所带来的RF的堆叠达到的近似的长程依赖, 这么做需要堆叠十分多层次导致优化困难、参数计算量变大、相邻层次依赖关系严重但是距离较远的层次依赖关系变轻等问题的存在. 本文的做法就是提出了一个本质是针对每个点的全局Attention机制, 就是文中的non-local module.
  3. 《Deformable Convolutional Networks》: 本文最主要的思想就是通过conv操作的采样位置的可变性使得感受野可以自适应的变化, 使得关注到应该关注的区域(PS:换个角度也可以看做是考虑到了额外的精度带来了一定的有效性), 对feature会开一个额外的分支去预测offset, 比较轻量. 对RoI Pooling也是改成了每个bin预测一个offset, 然后在RoI之外的地方去关注想要关注的区域的可形变RoI Pooling. 此外将segmentation领域说到的pool有中心不对齐的问题进行了引入介绍, 将ResNet中pool带来的中心不对准问题进行了一定的解决.
  4. 《GCNet》: 没有读这篇文章, 但是看过对这篇文章的简单陈述, 其是对Non-local和SENet的结合. 首先Non-local是对于每个像素都考虑global计算的的, 计算消耗比较大, 不是很划算. 本文通过实验发现了一个观点, 在最后的各个点上看, attention分布的区域都是相似的, 因此本文认为经过训练之后, 全局上下文并不是位置依赖的, 因此计算一个通用的全局的attention map来简化non-local block. 结构上结合了Non-local与SE Block的设计最后取得了不错的效果.
    《SENet》: 虽然本文是针对Classification提出的模块, 在没有引入很多计算量的同时带来了一定的性能提升, 本质是channel-wise的attention. 其通过对特征图的每个通道进行一定规则计算(文章为了简单对平均和max做的实验, 但还是可以用更复杂结构的), 然后进行FC的重组最后得到权重归一化之后进行特征的按照channel的加权. 但是在一些detection任务中还是很有效的.(比如M2Det)

十六、轻量级网络以及模块设计

  1. 《MobileNets: Efficient Convolutional Neural Networks for Mobile Vision Applications》: (只是v1) 本文是针对轻量级网络的设计, 目的是能够在移动设备上高速有效的运行, 本文是使用depth-wise separatable conv(深度可分离卷积)进行加速卷积操作的处理去构建轻量级的DNN网络. 本文的思想是提出了一类的模型架构, 其主要针对的是inference时间上的问题, 同时也会关注模型大小的影响. 本文的关键部分就是首先使用了depth-wise separatable conv (值得注意的是, (1). 计算量公式上来看(但是实际上计算量大头不在这里)因为现行的conv都是采用小kernel卷积, 所以时间加速比的瓶颈就出在这里啊, 按照3 3的卷积核来考虑, 大概加速的时间是8~9倍 (2). 传统的卷积操作会在conv之后接上一个BN层以及relu层, 深度可分离卷积模块则是先进行 平面上的卷积, 然后跟上一个BN和relu 之后再进行1 1conv之后再跟上一个BN和relu, 也就是将原本的一个conv模块拆成了两个模块). 计算量与参数量实际上都在1 1conv那里(从公式看则是认为计算瓶颈在卷积大小), 但是本文说使用GEMM可以很好的进行优化1 1conv计算. 另外就是本文为了进一步减少参数量以及计算量引入了两个超参数α, ρ, 分别表示width multiplier与Resolution multiplier. width multiplier指的是直接对原模型的输入输出通道数进行压缩, 参数、时间都是原来的α^2. Resolution multiplier指的就是对input image以及各个resolution的feature map进行一定比例的放缩, 计算量就是原来的ρ^2. 综合两者就是原来的α^2ρ^2.
    • PS: 看总结中提到了一个问题: 就是本文为什么强调Mobile, 其就是针对移动设备进行的模块设计, 移动设备是没有GPU的, CPU计算的时候缓存命中是十分重要的, 将两者分离用更多的模型层数达到一个高的缓存命中率, 带来的性能提升是巨大的. 而对于GPU来说层次的加深可能带来的时间损耗会比较多(参照ResNet看就知道了, 虽然VGG的参数量与计算量是远远大于ResNet的, 但是呢ResNet却比VGG要慢好多.)
    • PS: Depth-wise separatable conv的思想就是, 原本的卷积时图像所有区域的所有通道都被同时考虑了, 能不能把通道和时空区域分开考虑. Xception的思想, 先对图像的每个channel进行一个卷积操作, 然后得到新的feature map之后对这批通道的feature map进行1*1的conv操作. 两篇文章中都好像提到了这种深度可分离卷积的本质是Factorized Convolutional(Factorized Convolutional Neural Network文章, 简单看了一下总结, 本文就是介绍了各种各样的不使用传统的conv计算所有特征通道的值, 而是选择部分利用通道以及其他的一些方法, 本文所谓的深度可分离操作其实是这篇文章的一种极端的做法(保证输入输出通道相等, 同时卷积核开始的depth就是1, 每次只关注平面和深度两者之一的信息))的一种形式.
    • PS: 一个很有意思的现象, 个人认为既然已经将深度和平面分离开了, 那么后续继续在深度(channel上)进行一个SE module(channel-wise attention)的话应该还会得到一定程度上的性能提升. (PS: 好像已经工作这么做了呢…)

杂项(不好分类的Paper)

  1. 《Adaptive Feeding: Achieving Fast and Accurate Detections by Adaptively Combining Object Detectors》: 本文的主要思想就是选择性的将图片递给不同的网络以取得一个accuracy & speed 的 trade-off. 本文准备了两个detection-network, 一个快但是稍微不准确, 一个慢但是准确. 所以文章希望能对简单的容易进行detection的图片使用快的模型, 不容易识别的模型使用慢的模型. 为此文章对难和简单的定义是基于单张图片的mAP所决定的. 文章会先进行一个简单特别快的detector去初步的识别一些信息, 然后用这些信息通过SVM进行图片是简单还是难的分类, 然后选择性的将图片feed给相应的detector.
  2. 《A Unified Multi-scale Deep Convolutional Neural Network for Fast Object Detection》: 本文的主要观点就是 (1). 单一尺度因为感受野的统一而不能匹配各种尺度的object, Multi-scale会为了解决感受野大小好物体大小不一致的问题 (2). 使用feautre pyramid 而不是 image pyramid. 贡献主要是 (1). 总结了一下 几种为了匹配到多种scale的物体而提出的各种架构组合 (2). 本文选择使用多尺度, 多template进行进行detection.
  3. 《Learning Region Features for Object Detection》: 关注region特征提取的无handcraft参数的设计. 首先, 本文对传统的RoI特征提取部分进行了一个公式化的描述.定义了一个区域(这个区域可以是传统方法所定义的RoI的区域, 也可以是扩大到全特征图范围(比如deformable conv文章中提出的对RoI的变体就是可以移动的, 范围就是在全局的了)), 特征点的提取就是对该区域中的点在原特征图上的加权求和. 本文引入了两个改变可以影响weight的系数(两个attention) -> (1). 位置p和RoI box b之间的集合关系.对这个关系会用embedding之后进行可学习权重(就是卷积核等)变换处理之后进行inner product得到权重 (2). 使用image feature x. (文章直接用一个1 * 1conv去对feature map进行处理) 两个系数得到之后 相加再softmax一下就得到了权重. 之后就按照对应位置进行feature和权重的加权就可以了. 另外值得提一下的是, 将region区域放的越大性能越好, 但是相应的就会越慢, 可以认为是context带来的, 也许是计算量带来的….

涉及到多方面改进的论文的统一总结

  1. 《Libra R-CNN: Towards Balanced Learning for Object Detection》:((1). Loss改进 (2). anchor改进 (3). feature fusion改进) 本文实际上可以看做实践中的a bag of tricks. (1). 针对anchor的改进: 作者实验表明, 有超过60%的hard negatives与GT的overlap超过了0.05, 但是通常采取的随机sampling只能提供30%左右的超过0.05IoU的samples, 导致了hard samples被easy samples淹没造成了imbalance问题 -> 按照IoU区间进行anchor的概率采样以达到符合期望的IoU采样(将IoU划分为一定的bins中, 每个bins中取一定数量的hard negatives) (2). 针对feaute pyramid的改进, 将feature pyramid中的feature map resize到一个中间尺度上, 然后加权求和(文中是每一层的权重都是相同的), 然后将这个新feature map进行refine处理, 之后resize到原来的feature map的尺寸和原feature map进行相加, 利用这个新得到的feature pyramid进行进一步的detection (3). Loss改进: 作者提到直接增加localization loss会使得模型对outliers更加敏感, 也就是hard examples会产生很大的梯度, 远超过inliers(easy samples)所贡献的梯度, 对训练造成不好的影响. 增加了inliers的权重
  2. 《DSSD : Deconvolutional Single Shot Detector》: ((1). Feature fusion改进 (2). encoder-decoder结构) : 本文借鉴了hourglass结构, 两个核心贡献就是 (1). 使用功能更强的resnet-101作为backbone代替VGG (2). 利用Encoder-Decoder漏斗结构去进行context融合, 提升小物体检测的准确率. 除此之外本文还对feature fusion以及detection模块进行了一定的改进.
  3. 《FSAF:Feature Selective Anchor-Free Module for Single-Shot Object Detection》: ((1). anchor-free (2). 训练策略) (1). anchor-free: 本文实际上只是不是完全意义上的anchor-free, 本文结构采用的retinaNet, 其只是在进行anchor-based进行class prediction, bbox regression的过程的4个conv层的后面额外添加了一个anchor-free分支, 这个anchor-free分支是不进行anchor划分的, 所以输出通道是anchor-based分支anchor数分之倍. anchor-free进行预测的是点距离”bbox”的左、右、上、下四条边的距离(PS: 和CenterNet很像, 但是前面不同) 本文单独使用anchor-free分支的时候效果要比使用anchor-based要差, 但是可以与anchor-based分支结合使用达到更好的效果 (2). 训练策略: 本文通过在训练过程中对每个instance进行anchor-free分支的forward一遍得到各个尺度上的loss, 然后选择loss最小的分支进行backpropagation, 本文认为这么选择每次都会刷新loss的lower-bound. PS: 本文对anchor-free分支的训练是使用focal-loss处理分类结果 IoU loss处理”bbox regression”的
  4. 《Receptive Field Block Net for Accurate and Fast Object Detection》: ((1). feature fusion module (2). 感受野) RFBNet, 本文主要是从感受野的角度出发, 和deconv(强调感受野的变化性)和dilated conv(扩大感受野)不同, 本文同样是在感受野上面做文章, 但是本文强调的是eccentricity(很多地方译为: 偏心度), 要求(1). 中心的感受野的kernel要小, 边缘感受也到的kernel要相对大 (2). 感受野要覆盖到整个区间. 做法就是模块内开多个分支, 使用不同大小的diconv去进行处理, 然后concate起来接上一个1*1conv.
  5. 《CBNet: A Novel Composite Backbone Network Architecture for Object Detection》: 本文的想法是很简单的, 就是希望能够不重新设计新的针对detection的backbone, 不进行预训练操作的情况下, 牺牲一定的计算去得到一个性能更好的backbone. 本文通过将几个相同结构的SOTA的backbone进行”拼接”, 将前一个backbone的feature map与下一个backbone的feature map进行fusion, 直到最后一个. 文章实验是堆叠到了三个, 性能提升越来越少, 同时fusion不同的scale feature map也提出了几种不同的方式. 值得说的是在本文中其并不会过度的影响backbone时间的损耗(因为backbone在two-stage上的时间消耗并不主要在backbone上), 但是参数量的增加是显然的.
  6. 《Cascade R-CNN: Delving into High Quality Object Detection》: 本文针对RPN结构进行改进, 认为原本的使用一次RPN的做法在使用低IoU threshold的时候通常会导致noisy detection的产生, 使用高IoU threshold则会导致performance的退化, 本文做法就是使用一系列的有着逐渐升高的IoU的RPN, 逐步减少false positive的出现. 同时本文做实验表明了使用指定的IoU进行训练将会导致对指定的quality level(GT IoU大小)达到最优, 而对于其他的quality level是次优的, 所以实际上的本质出发点就是获取各个quality level的最优. 针对iterative RPN作者认为存在的问题是: (1). 只使用了一个相同的threshold, 所以只是对某个quality level的最优, 导致了其他quality level性能的退化, (2). bbox的distribution随着RPN阶段的处理之后分布发生了明显的变化(另一篇文章认为GT bbox的分布应该是delta分布, RPN的目的是将可能的anchor分布尽可能的向这个狄拉克分布靠近, 所以每个阶段之后分布范围都会缩小, 正如实验图所示), 对前一阶段RPN最优的设置可能导致是次优的, 所以用了一系列的其他工程方法进行解决, Cascade R-CNN则只需要改变GT IoU训练的阈值.
