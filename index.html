Simple Summary for (Chapter15--Exercise) 简单总结(VGG) Very Deep Convolutional Networks For Large-Scale Image Recognition

Posted on 2020-01-01

(Chapter15–Exercise) 简单总结(VGG) Very Deep Convolutional Networks For Large-Scale Image Recognition

Introduction:

  1. 本文的主要贡献: 验证使用 3*3 conv 的网络结构加深达到的效果
  2. 强调了之前的针对 (Krizhevsky的) Conv 网络的改进是 (1). 在第一个 conv 层使用更小的 conv 以及 stride. (2). 使用多尺度的方法进行network的训练以及测试(这里指的是 Overfeat, 和一些其他的当年的作品使用的方法)
  3. 提到了本文想要解决的是ConvNet结构设计的另外一个重要的方面: 深度 -> 将架构中所有的其他参数都固定, 然后仅仅去增加网络的深度, 堆叠更多的 conv layers. 认为是使用了小的 3 * 3 的卷积使这种做法变得可行.
  4. 本文贡献: 提出了一个新的更加准确的SOTA的深度convnet, 具有很好的广泛应用性, 且其提取到的特征使用简单的pipeline也可以得到一个很好的效果.

Method:

  1. 架构的一些设计:
    • (1). 224*224 RGB input image
    • (2). preprocessing -> 简单的减去平均图而已
    • (3). 使用small RF conv(3 3甚至是1 1的)
    • (4). 卷积前后保持尺度不变
    • (5). 尺度大小变化通过2 * 2max-pooling来做.
    • (6). 后续会使用3个FC层(4096-4096-1000)
    • (7). 所有的hidden layers都会使用ReLU进行激活
  2. 强调和主流数据集的主流设定不同的是, 本文提出的结构没有在第一个conv层使用大的kernel或者打的stride.
  3. 主张使用多个 3 3 的小的conv去在代替大的如 5 5、7 * 7 的conv. 认为好处如下:
    • (1). 多个卷积核后面跟上了多个激活单元, 意味着非线性的增加, 做出的决定是更加有辨别能力的.
    • (2). 减少了模型的参数
  4. 提到了适量插入 1 1 conv的目的是为了增加非线性的同时并不改变 Receptive Field. (提到了 1 1 conv是在 Network in Network中广泛使用的)
  5. 关于训练的一些设计:
    • (1). 使用带动量的mini-batch gradient descent进行训练, 动量为 0.9
    • (2). batch-size 调整到256
    • (3). 使用 L2 正则去进行权重约束
    • (4). 使用dropout regularisation(对前两个FC层)
    • learning rate 开始设置为 10^-2, 然后当在validation上的准确率不在增长的时候就减少十倍, 训练370K iteration(74 epochs)之后停止 -> 文章说本文的网络深参数多, 但是能够更快的收敛的原因是因为1). 网络的深度以及小conv kernel的使用带来的隐性约束 2). 特定层的预先初始化 -> 作者先使用一个比较浅的网络去进行训练, 然后提取这个网络的浅层(前4层以及后3层conv) 中间的使用随机初始化进行(PS: 同时训练的时候并不会减少对这些层的学习率) (PS: 作者提到了, 可以不按照这个方法, 而且是使用 Glorot&Bengio 提出的随机初始化方法代替这个方法也可以达到好的效果)
  6. Training image size:
    • PS: 设 S 作为各向同性放缩的最短边长度
    • (1). 如果将crop size设置为固定的 224 * 224 的时候, S 是要不小于 224 的, 当 S = 224 的时候, crop将会crop整个image. S >> 224 的时候只会 crop 出来 image 的很小的一个部分, 这个部分将会包含一个小物体或者一个物体的部分.
    • (2). 作者考虑两种进行 training scale S的设定: 1). 固定 S -> single-scale training. 2). 设置 S 为多尺度训练, 将训练图片随机放缩为 [S_min, S_max]之间
  7. Testing: 提到了为了不再对每个crop出来的image patch(因为之前前面是FC层)进行重复的的计算然后ensemble, 选择先对整个放缩好的 image 先进行特征的提取(7 7 conv + 2个1 1 conv) 得到一个 class score map(通道数是和类别数相等的, 并且是可变的空间分辨率的) -> 为了得到一个固定大小的关于image的class scores, 会先进性spatially averaged. 然后再用FC层去进行处理.
    • 但是后续还是会选择 crop 去提高 performance

Exercise 问题的一些回答:

  1. 为什么变深 -> (1). VGG 中提到是看到了很多的效果好的工作的趋势都是 加深网络 然后使用 小卷积, 所以作者考虑使用 深度作为一个变量参数 通过加深网络去尝试加强网络的性能
  2. 变深的问题 -> (1). VGG 中: 加深会导致网络参数的增加 -> 1). 使用 小卷积代替大卷积 减少参数量同时增加非线性 2). 参数难以收敛 -> 小卷积减少参数&&对特定的层次使用初始化预训练简化训练过程(但是后面说按照后续的参数随机初始化做法就可以提高性能超过进行预训练初始化得到的结果) (2). 参数以及时间的增加 -> 1). 使用 1 1 conv 在 3 3 / 5 * 5 conv 前面降低通道数目

Simple Summary for Grad-CAM: Visual Explanations from Deep Networks via Gradient-based Localization

Posted on 2019-12-27

《Grad-CAM: Visual Explanations from Deep Networks via Gradient-based Localization》

Introduction:

  1. 本文的目的: 提出一个方法去针对于CNN-based models的可视化解释, 使网络模型结构更加透明清晰(可解释)
  2. 本文的方法 Grad-CAM(Gradient-weights of Class Activation Mapping) 使用任意 target concept 的梯度, 然后流淌到 final convolutional layer去产生 粗略的 localization map 去 highlight 图片中对这个 concept 的重要的部分
  3. 提到了和之前的方法不同, Grad-CAM对广泛的 CNN model-families 都是可行的, 包括了:
    • (1). 包含FC层的VGG
    • (2). 用于产生结构化输出的 CNN
    • (3). 包含多模态输入或者强化学习但是不包含网络结构的改变和re-training的CNN模型(我翻译的好像有点问题, 应该指的是这个方法(Grad-CAM)不用改变网络结构或者重新训练整个网络)
  4. 文章将 Grad-CAM 和现有的细粒度的 visualization 结合去生成高分辨率的类别区分的可视化 Guided Grad-CAM 并将其应用到 image classification, image captioning, visual question answering models 等
  5. 文章说对于 image classification model 的 context, 本文的可视化能够
    • (1). 给model失败的模式提供内在的知道
    • (2). 比传统的方法效果要好很多
    • (3). 更能够方便理解model
    • (4). 可以帮助model通过对不同的 dataset 得到的 dataset bias 进行泛化能力的提高
  6. 对于 non-attention based models, 对非 attention-based 的模型也可以 localize inputs
  7. 摘要最后部分强调了一下自己模型的可解释性很强, 能够将好/坏的模型很明显的区分开
  8. 提到了 CAM (Class Activation Mapping) -> 用于不包含 FC 层的 classification CNNs, 提到了实质上那篇文章对 model 的复杂度和性能进行了 trade-off, 以追求更多的模型工作的透明度. 而本文Grad-CAM相反, 本文方法可以使得 SOTA 的 model 可解释之外不对他们的结构进行修改, 避免了可解释性和准确性的 trade-off. 本文方法是 CAM 的泛化, 并且可以应用到很广泛的CNN家族模型上
  9. 本文对 good visual explanation 的解释:
    • (1). class-discriminative: 能够定位该类别在这个image中的重要位置
    • (2). high-resolution: 能够捕捉到fine-grained details
    • PS: 举例中提到了, Guided Back-propogation 和 Deconvolution是 high-resolution 的, 能够 highlight 图片中的 fine-grained details, 但是不是 class-discriminative 的(因为对不同类别的图片来说并没有很明显的区别). 而 CAM 或者本文的 Grad-CAM 就是 class-discriminative
  10. 作者希望能得到一个更好的结果, 所以选择将 Guided Back-propogation 和 CAM 方法进行结合, 从而得到既是 high-resolution 又是 class-discriminative 的方法 -> Guided Grad-CAM
  11. 本文的贡献:
    • (1). 提出了 Grad-CAM, 其具有 class-discriminative localization 的性质, 并且能够对任意的 CNN-based network 进行 visual explanations, 而不需要改变网络的结构也不用对网络进行重新训练
    • (2). 能够使用 Grad-CAM 对 top-performing的 classification, captioning, VQA 模型进行使用, 可以达到多种预期的作用
    • (3). 在 classification 任务使用 ResNets 去进行可视化探究, 提到了从 deep 到 shallow layers, 当使用的 layers 具有不同的 output dimensionality 的时候 Grad-CAM 的辨别能力显著降低.
  12. Related Work 中提到了
    • (1). 关于 visulaizing CNN 来说, 很多先前的工作都是通过 highlight “import pixels” 去进行的. 提到了 1).[40] 那篇文章将 预测类别分数的偏导进行可视化 2).Guided Backpropogation 和 Deconvolution 则对原本的 “raw” gradients 进行改变从而得到质量上的提升???. 其他的一些可视化方法综合多张 images 去最大化的激活网络的 unit, 或者 invert a latent representation(翻转潜在的表达???). 虽然这些是 high-resolution 的 并且是 class-discriminative 的, 但是其是对 model 总体上的可视化而不是对特定 input image 的可视化.
    • (2). Assessing Model Trust: 略
    • (3). Weakly supervised localization: 提到了一个类似的方向, 就是弱监督的定位CNN中的context(仅仅通过给定的图片和图片整体的类别进行图片中物体的定位)
    • PS: 提到了 CAM 是会改变 CNN 网络结构的, 其会使用 conv 或综合 global average pooling 层代替 FC 层. 提到了 CAM 的副作用就是需要 feature maps 去直接接在 softmax layers 上, 所以只对使用global average pooling 的模型有效 (i.e. conv feature maps -> global average pooling -> softmax layer 这样的结构要求对于很多的模型来说是次优的, 同时对于很多问题来说是无法解决的 -> 本文提出了一个新的方法, 将 feature maps 和gradient signal进行结合, 所以不再需要对网络结构进行改变.)
    • PS: 当在 fully-convolutional 结构的 网络之中, Grad-CAM 就退化成了 CAM. 因此作者认为 Grad-CAM 可以看做 CAM 的泛化.
    • (4). 提到了一些方法会对 input image 扰动后的图片进行分类来实现 localization.

Method:

  1. 很多先前的工作都断言在CNN高层次结构上会包含更加深层次的表达. 同时提到了 conv 能够自然地保留 FC 层所忽略/损失掉的 空间信息 -> 所以我们可以期待最后一层 conv layer 能够对 high-level semantic 和 detailed spatial information进行最好的折衷. 这些层的 neurons 在 image 中寻找 class-specific 的语义信息.
  2. Grad-CAM 使用流入 CNN 最后一个 convolutional layer 的 梯度信息去理解每个 neuron 对于特定类别(文章中说的是 interest )的决策的重要程度.
  3. 为了获取到 class-discriminative 的 localization map,
    • (1). 首先计算特定类别 c(指的就是选定类别)的梯度, 到 conv 层的 feature map $A^k$ 上就有梯度 $\frac{δy^c}{δ_{A^k}}$ 这些梯度回流之后将会使用 global average-pooled 去获取 neuron importance weights $α_k^c = \frac{1}{Z}*Σ_iΣ_j \frac{δy^c}{δA_ij^k}$
    • (2). 进一步的会使用这个 $α_k^c$ 对 $A^k$ 进行加权, 然后使用 ReLU 进行激活得到 最后的 class-discriminative 的 localization map.
  4. 提到了需要注意的是在比如 VGG/AlexNet 等网络中最后一个层次的 conv 是很小 resolution 的.
  5. Grad-CAM as a generalization to CAM:
    • 回顾了一下 CAM 方法, 其会将最后的 feature map 首先进行一个 global average pooling 层 得到一系列的 c_i (i 对应于 feature map 的 i 通道)分数. 之后会使用 c_i 获取一系列的 w_1 ~ w_n(PS: 各个地方都说这些w是softmax层的对应类别不同层的参数, 个人理解是所说的softmax层其实是FC+softmax, 然后使用的parameters就是FC层的参数, 总体流程就是 feature map -> GAP(global average pooling) -> FC -> softmax. 然后加权的话我们先 pooling 后拿到了 c_i, 然后再提取 FC 中的针对特定类别的 w_i, 用w_i 对 c_i 进行加权, 得到的值乘以 feature map 对应的 i channel 最后求和得到的就是 heatmap 了) 进行线性变换再次加权 然后求和就得到了最后的 localization map 了. -> 再次提到了因为不是每个模型都有 class feature weights(指的是不是每个模型都是这么干的, 比如有的模型后面可能没有全连接层 eg: 各种 one-stage 方法), 所以需要改变网络的结构并且重新训练. 而本文提出的 Grad 方法并不用
    • 仔细想一下, 当 Grad-CAM 针对于 CAM 所要求的结构的时候对 FC 层次进行求导, 实际上拿到的 Grad-CAM 的 $α_k^c$ 就是 CAM 中的 $w_k^c$
  6. Guided Grad-CAM: 上面的 Grad-CAM 得到的 localization map 还是和最后一个 conv 相同大小的, 不是 fine-grain 的, 所以… 把 Guided Back-propagation 和 本方法进行融合 -> 细粒度的 localization map. 就是反向传播出来 Guided Back-propogation 之后, 用 Grad-CAM 得到的 localization 进行点乘加权即可.

Simple Summary for Residual Bi-Fusion Feature Pyramid Network for Accurate Single-shot Object Detection

Posted on 2019-12-21

简单总结《Residual Bi-Fusion Feature Pyramid Network for Accurate Single-shot Object Detection》

Introduction:

  1. 本文提出了 ReBiF (Residual BiFusion) Feature Pyramid. -> 同时融合深层次和浅层次的 features -> 高效且鲁棒的 object detection.
  2. RBF feature pyramid 的构成是简单, 轻量级的, 只用临近的 3 层被用来去 fused 通过 简单的concatenations 和 1*1 conv 去构成 prediction map.
  3. 本文主要贡献总结:
    • (1). 新型的 residual bi-fusion feature pyramid -> 融合浅层和深层的 features 以追求更精准的 one-shot object detection.
    • (2). 新的 feautre pyramid 可以简单的被训练和集成到不同的 backbone 中 (因为 residual 的特性)
    • (3). 本文的 residual feature pyramid 对不同的 datasets, object sizes, object classes 有着很好的泛化性
    • (4). 可以减少 pooling 带来的位移不变性给object detection的影响
    • (5). 在 VOC 和 MS COCO 数据集上达到了 SOTA 的结果

Method:

  1. 首先还是对 FPN 进行了一个简单的描述, 然后再次强调其缺陷 -> 因为 pooling 和quantization 的问题, 很难保留准确的位置信息, 为了解决问题, 从 shallow feature map 也获取信息可以有效的避免小物体的漏检.
  2. 提到了目前的 SOTA 的 bi-directional 方法都是从原图或者低层次的 feature layers 去构建新的 feature streams 以精准定位. 但是这么做 计算量以及参数量是很大的.
  3. 本文提出的做法包括
    • (1). CORE(Concatenation and Reorganization) 文章 Fig1. b)所示结构 -> 相邻的三个层次进行 feature map 的 fusion (直观来说就是语义信息(high level)和位置信息(shallow layer)的融合) -> 因为只是相邻三层的 re-using 机制, 所以相比来说能够节省很多的 memory 和 bandwidth.
    • (2). 对 (1). 中 CORE 模块的 output 进行进一步的 purified 去生成语义信息更加丰富的 features. Fig1. c)
    • (3). 结合 residual 机制, 前一阶段的 RECORE(应该实际上就是CORE, 只是名字不同的问题) 将会用一个 shortcut 连接到下一个阶段
  4. CORE Module:
    • (1). 提到了想要同时融合 deep 和 shallow 的layer的semantic feautres, 但是又不想引入过多的 dithering operation???(eg: conv) 和 computationally expensive 的 操作(eg: pooing, addiction)并且尽可能的去保留预测的信息, CORE module 使用 concatenation 去进行 features 的 fusion.
    • (2). 同时 concatenation 也不仅仅是像原来的方法简单的拼接全部层次的信息, 而是recursively concatenate临近层次的features. -> 可以看结构图知道, 是上下(deeper && shallower)层 分别 经过 Bottleneck 结构 和 upsample/reorg 操作得到和当前层次相同大小的 feature map, 然后和当前层次的feature map进行拼接
  5. Purification:
    • (1). 将 CORE Module 输出的信息进一步的通过 Purification Module 进行处理, 文章中给出的结构是 Bottleneck -> conv -> Bottleneck -> conv -> Bottleneck
    • (2). 值得注意的是, 文章中是第二个 Bottleneck 输出的结果作为下一个阶段的 CORE Module 的输入, 而不是 CORE Module 最后的输出结果进行 fuse 操作.
  6. BiFusion Feature Pyramid:
    • (1). 简单的复述了之前的几个观点而已… 具体结构就是 Figure.1 c) 所示的, 只是说明自己为什么快, 节省了内存和时间
    • (2). PS: 没有觉得有什么很新颖的东西, 双边信息融合之前的做法并没有看到过…
  7. Residual Feature Pyramid:
    • PS: 果然 RECORE 就是对 CORE 改了个名字, 应该是因为结合了上个 CORE Module 的输出, 所以叫做 RECORE 了
    • (1). 具体细节见 论文中的 Figure5.
    • (2). 大概描述就是 前一阶段的 CORE 的输出会用一个 shortcut 直接和下一个阶段的 CORE 进行 dot add.
  8. Improvement on Large and Medium Object Detection:
    • (1). 提到了, YOLO 在小物体的性能上有提升, 但是牺牲了在大物体上的精准程度. 为了避免这种情况, IPIRC 2019 的冠军 选择提高大物体和小物体的检测上 的性能, 但是减少小物体上的性能来提升总体性能.
    • (2). 本文为了同时提升小物体和大物体上的性能, 选择在 residual pyramid 结构之后再次通过一个 bottom-up 的 pathway 去进行 进一步的feature fusion (BFM 结构)

    • (2). 两个没有被关注过的问题: 1). attention bias 2). feature selection
    • (3). attention bias -> 有着很好的 views 的物体将会倾向于获得到 detector 的更多的关注, 使得其他的物体容易被忽略(因为训练过程中的anchor-points misalignment).

  1. Preliminary on Anchor-Point Detectors: 首先对 anchor-point-based baseline(作者上一个作品FSAF) 进行了回顾: 这部分回去看 FSAF 就好
  2. Soft-Weighted Anchor Points:
    • (1). Attention bias: 强调了当面临着类似于 occlusion, cluttered background 等具有挑战的情况下, vanilla anchor-point detector 会存在 attention bias 问题. -> 有着很清晰的表现或者引人注目的表现的物体通常来说都是会产生一个很高的分数, 抑制了周围的物体, 甚至可能覆盖掉周围物体的表达. -> 导致了过分关注 foreground instances 抑制了 background instances.
    • (2). 问题的原因: 认为是因为 在靠近物体instance边界的位置上因为feature misalignment所导致的位置不确定使得边界周围的值 不必要的过高 所导致的. -> 所以在物体边界的其他的 anchor-point 会因为没有足够的与其对应的 instance 所以导致效果不好.

    • (4). 因此不应该将物体 center 附近和 boundary 附近同等程度的对待 -> 走 CenterNet 的思路了… 看上去好像没什么区别… (PS: 细说的话还是有的, 其自由度相比来说更强, 但是可能会牺牲一些东西…)

      • 1). 主要思想: 对每个 anchor point 都给一个权重, 对每个 positive anchor point, 这个权重将会依赖于 其 image space location 与 instance 中心 的距离.
      • 2). 这个距离越长, 权重越低
      • 3). 对于 negative anchor point 来说, 所有权重都还是 1 不会变化

      • 1). 首先要根据 feautre response 的 pattern 进行选择. -> 认为instance-dependent loss 是一个很好的反应一个物体应该属于哪一个层次的东西.(FSAF思路就是这个样子)
      • 2). 我们应该允许来自于多个尺度的 feautres 在训练和测试中对每个 instance 产生作用. (PS: FoveaBox表明了将instance分配给多个 feautre level 是有利于提升性能的.)但是强调了如果将物体分配到了太多的层次也是会严重影响性能的. -> 作者认为是 hard feature level selection 所导致的. -> 作者提出的就是能够根据 instance 去进行每个 pyramid levels 的 reweight.
    • (4). 解决办法: 选择去训练一个 meta-selection 网络去预测 soft feature selection.
      • 1). 首先模块的输入是从各个 pyramid levels 中提取出来的 instance-dependent feature responses (文章中是通过 RoIAlign layer 进行提取的) (PS: 我之前也想过这么干, 但是对于anchor-based one-stage detector来说那个东西真的叫做detector吗…)

      • 3). meta-selection模块 -> 3 个 3 * 3 conv层(无padding, 跟ReLU). 然后跟上一个 FC 层做 softmax(输出与通道数相同). 使用 cross entropy loss 去进行优化, label是根据 FSAF 最小的 loss 分支决定的 one-hot vector 进行优化的.
      • 4). 因此每个 instance B 都会有一个 权重(meta-selection的输出) 再结合上上一个的 soft-weight anchor-point, 就可以得到对应最终的该instance的权重.
      • 5). 网络的总Loss会包含两个部分, meta-net 部分的 loss 的 weight 是会调整的.

  1. 本文的主要贡献: 验证使用 3*3 conv 的网络结构加深达到的效果
  2. 强调了之前的针对 (Krizhevsky的) Conv 网络的改进是 (1). 在第一个 conv 层使用更小的 conv 以及 stride. (2). 使用多尺度的方法进行network的训练以及测试(这里指的是 Overfeat, 和一些其他的当年的作品使用的方法)
  3. 提到了本文想要解决的是ConvNet结构设计的另外一个重要的方面: 深度 -> 将架构中所有的其他参数都固定, 然后仅仅去增加网络的深度, 堆叠更多的 conv layers. 认为是使用了小的 3 * 3 的卷积使这种做法变得可行.
  4. 本文贡献: 提出了一个新的更加准确的SOTA的深度convnet, 具有很好的广泛应用性, 且其提取到的特征使用简单的pipeline也可以得到一个很好的效果.

Method:

  1. 架构的一些设计:
    • (1). 224*224 RGB input image
    • (2). preprocessing -> 简单的减去平均图而已
    • (3). 使用small RF conv(3 3甚至是1 1的)
    • (4). 卷积前后保持尺度不变
    • (5). 尺度大小变化通过2 * 2max-pooling来做.
    • (6). 后续会使用3个FC层(4096-4096-1000)
    • (7). 所有的hidden layers都会使用ReLU进行激活
  2. 强调和主流数据集的主流设定不同的是, 本文提出的结构没有在第一个conv层使用大的kernel或者打的stride.
  3. 主张使用多个 3 3 的小的conv去在代替大的如 5 5、7 * 7 的conv. 认为好处如下:
    • (1). 多个卷积核后面跟上了多个激活单元, 意味着非线性的增加, 做出的决定是更加有辨别能力的.
    • (2). 减少了模型的参数
  4. 提到了适量插入 1 1 conv的目的是为了增加非线性的同时并不改变 Receptive Field. (提到了 1 1 conv是在 Network in Network中广泛使用的)
  5. 关于训练的一些设计:
    • (1). 使用带动量的mini-batch gradient descent进行训练, 动量为 0.9
    • (2). batch-size 调整到256
    • (3). 使用 L2 正则去进行权重约束
    • (4). 使用dropout regularisation(对前两个FC层)
    • learning rate 开始设置为 10^-2, 然后当在validation上的准确率不在增长的时候就减少十倍, 训练370K iteration(74 epochs)之后停止 -> 文章说本文的网络深参数多, 但是能够更快的收敛的原因是因为1). 网络的深度以及小conv kernel的使用带来的隐性约束 2). 特定层的预先初始化 -> 作者先使用一个比较浅的网络去进行训练, 然后提取这个网络的浅层(前4层以及后3层conv) 中间的使用随机初始化进行(PS: 同时训练的时候并不会减少对这些层的学习率) (PS: 作者提到了, 可以不按照这个方法, 而且是使用 Glorot&Bengio 提出的随机初始化方法代替这个方法也可以达到好的效果)
  6. Training image size:
    • PS: 设 S 作为各向同性放缩的最短边长度
    • (1). 如果将crop size设置为固定的 224 * 224 的时候, S 是要不小于 224 的, 当 S = 224 的时候, crop将会crop整个image. S >> 224 的时候只会 crop 出来 image 的很小的一个部分, 这个部分将会包含一个小物体或者一个物体的部分.
    • (2). 作者考虑两种进行 training scale S的设定: 1). 固定 S -> single-scale training. 2). 设置 S 为多尺度训练, 将训练图片随机放缩为 [S_min, S_max]之间
  7. Testing: 提到了为了不再对每个crop出来的image patch(因为之前前面是FC层)进行重复的的计算然后ensemble, 选择先对整个放缩好的 image 先进行特征的提取(7 7 conv + 2个1 1 conv) 得到一个 class score map(通道数是和类别数相等的, 并且是可变的空间分辨率的) -> 为了得到一个固定大小的关于image的class scores, 会先进性spatially averaged. 然后再用FC层去进行处理.
    • 但是后续还是会选择 crop 去提高 performance

(六)、简单总结mmdetection源码TwoStageDetector的流程细节(五) – 解析BBox head forward 以及相关 Loss

一、前言

  1. 上一篇总结介绍了关于 GT assign 以及 Proposal Sample 的整体流程, 这个部分就是 TwoStageDetector.forward_train 的最后的部分了(不考虑 mask 相关部分), 但是到这里还是有疑问没有解决的, 具体疑问看 3.
  2. 本篇总结将会对 BBox head forward 以及相关 Loss 部分(mmdetection代码的注释是这么写的)进行解析.
  3. 疑问就是从上一个总结就有的, 关于多尺度信息在训练的时候怎么去区分, 因为上一篇文章也注意到了, 进行 GT BBox 匹配的时候也并没有区分什么尺度问题, 都是根据 IoU 在进行的计算, 然后进行筛选, 仅由的针对多阶段的过程还是 RPN 部分的 rpn_head.get_bboxes 其将多尺度的信息直接并成了一个维度, 只是有一个选项是选择是否进行跨尺度的 nms 操作. 希望在本文的 roi_extractor 部分会涉及到如何进行尺度分配的部分
    • 问题找到答案了: 在 mmdet.models.roi_extractors.single_level.py 中的 SingleRoIExtractor 类的 forward 过程中 会进行 map_roi_levels 函数的处理, 会根据 RoI 的大小去分配对应的 stage
  4. 另外的一个疑问: 从这部分的代码中已经可以看出了, 提取出来的 features 是来自于各种不同 scale 的啊, 后续的分支是公用的吗???

二、总体流程

  1. 首先将上一个阶段得到的 sampling_results 中的每一个 img 的 SamplingResult 对象提取出来其中的具体信息( torch.cat([self.pos_bboxes, self.neg_bboxes]) ), 然后进行 bbox2roi 操作, 就是添加了一下所属图片的信息
  2. 重点: 使用 bbox_roi_extractor 进行每个 roi 的特征信息的提取
  3. 然后将得到的 features feed 给 bbox_head (这里包含了 classification 以及 bbox regression)
  4. 最后将 loss 记录一下就可以了
  5. 然后整个 train_forward 部分就结束了, 将 losses 返回就好了 (nice!!!)

三、细节描述

PS: bbox2roi 操作就不仔细陈述了, 很简单的操作

  1. 首先我们就应该看一下 bbox_roi_extractor, 这个东西在 __init__ 中好像就已经被我们注册了, 和其他的部分一样, 都是通过 build_from_cfg 进行注册, 这个部分是使用 ROI_EXTRACTORS 这个 Registry 进行的. 同时根据 config 可以知道, 使用的是 SingleRoIExtractor, 那么就去看一下这个类: PS: 好像就只有这一个 Extractor 233
    • (1). 还是先分析一下其 __init__ 函数: __init__(self, roi_layer, out_channels, featmap_strides, finest_scale=56):
      • 1). roi_layer: 关于 RoI layer 的 config 信息
      • 2). out_channels: RoI layer 输出的通道数
      • 3). featmap_strides: 输入进来的 feature map 的 strides list
      • 4). finest_scale: mapping 到 level 0 的 threshold (??? 存疑)
      • 整个函数中基本就是参数的注册, 但是有一行要注意 -> 在里面注册了 RoI layer (… 禁止套娃!!!(滑稽)), 所以我们要关注 RoI Layer 了呢…
      • 根据配置, 我们可以发现, 使用的是 RoI Align (RoI Pooling 已经被抛弃了 2333)
      • em… RoIAlign 藏在了 mmdet.ops.roi_align.roi_align.RoIAlign… 还是从 __init__进行分析: 涉及到的四个参数: 1. out_size: 输出尺寸(config 中设置是 7) 2. spatial_scale: 空间尺度 (PS: config中没有这个参数, 是在 build_roi_layers 中根据 featmap_strides 填入的, 每个 scale 都会对应一个 RoIAlign 层) 3. sample_num: 采样点数目 4. use_torchvision: 是否使用 torchvision … (默认是 False, 话说为啥不用…). 至于 __init__ 干了什么, 就基本啥都没干, 就是进行了一下赋值操作
      • 看 forward 函数把: 参数是 features 和 rois, 如果 use_torchvision 为 True, 那么使用 torchvision.ops.roi_align, use_torchvision 为 False, 那么使用 mmdetection 定义的 roi_align. 看一下自定义的 roi_align 理清一下逻辑(应该是提供了一个 cuda 加速版本…)
      • roi_align 函数的定义是 roi_align = RoIAlignFunction.apply, 即通过自定义实现的 RoIAlignFunction 类进行的. 看一下这个类的 forward, em… 实际上是要调用 roi_align_cuda 也就是 cuda 加速版本… 我 mac 上没有 cuda 所以就安装不了这部分…
      • 看看 torchvision.ops.roi_align: 两者套路完全一致… 都是用 C 去写的核心部分 (貌似是因为涉及到了 CUDA 加速计算… 好吧 去康康, 暂时略过了… 只是大概明白了什么操作, 是要对每一个 scale 都要进行 RoIAlign 操作, 见前言的3.)
      • 这就不用再分析 ROI layer 了, 继续看 SingleRoIExtractor
    • (2). 看看 forward 部分:
      • 关于输入: feats: 就是 feature map rois: 就是 sample 出来的 RoIs 信息 roi_scale_factor: (默认值是False) 表示收对 RoI 进行 Rescale
      • 1). 首先如果 feats (也就是各个 stage 的 feature map) channel 数目只有 1(就是不使用 FPN 的 Faster RCNN) 的时候, 就直接根据仅有的 RoILayer 进行处理然后返回即可 即 return self.roi_layers[0](feats[0], rois), 但是如果使用 FPN (即 channel 数目 > 1)那么就要进行下面的操作了
      • 2). 首先会获取一些维度信息, 然后使用 map_roi_levels 对 RoI 进行 stage 的分配 (就是这里, 就是这里, 这里将不同的 RoI 按照大小分配到不同的 stage 上), 康一康 map_roi_levels:
        • map_roi_levels(self, rois, num_levels): rois 就是所有的 RoI 框. num_levels: 使用的 feautre map stage 的数目
        • 具体操作就是: 先算一下每个 proposal 的面积, 然后除以 finest_scale 之后进行一下 log2, 取一下floor, 之后 clamp 到 0 - 3(这个 config 来看是 3) 之间. 然后返回就可以
      • 3). 创建一个 roi_num * out_channel * out_size * out_size 的 Tensor 用来装最后的提取出来的 feature
      • 4). 对各个 stage 的 roi_layers 传入对应的 feature map 以及 进行了 assign 到当前 stage 的 rois 去得到提取到的 features. 得到 RoI Features, 将 RoI Features 填充到对应的地方
      • 5). 返回这个 roi_feats
  2. 后面涉及到了一个 with_shared_head 参数, 看了一下框架的代码, 只有一个 shared_head 还是 res_layer 看上去也就是这个就是用来进行额外的处理的, 就是对 提取出来的 feature map 的信息进行进一步处理之后再传递给 bbox_head. 根据配置文件可以看出, Faster RCNN 中使用的是 SharedFCBBoxHead, 查找 SharedFCBBoxHead 之后可以发现, 其有一个父类 ConvFCBBoxHead, 并且 SharedFCBBoxHead 的 __init__ 函数是直接调用父类的 __init__ 函数的, em… 也可以发现 ConvFCBBoxHead 也有父类 BBoxHead. 让我们先分析一下这两个类的 __init__ 函数吧:
    • (1). SharedFCBBoxHead.__init__: 首先看一下这个类中的注释, 描述了这个类的流程是什么
    •                                              /-> cls convs -> cls fcs -> cls
    • shared convs -> shared fcs
    •                                              \-> reg convs -> reg fcs -> reg
    • 也就是我们首先会经过 classfication 与 bbox regression 共同要使用的 conv, fc 层, 然后才拆分为两个分支, 分别进行两个任务的 conv, fc 层, 最后得到结果
    • 所以有了这个目的, 让我们看一下其中的参数吧, 但是还是最好先看一下其父类 BBoxHead 的参数:
    • (2). BBoxHead.__init__(self, with_avg_pool=False, with_cls=True, with_reg=True, roi_feat_size=7, in_channels=256, num_classes=81, target_means=[0., 0., 0., 0.], target_stds=[0.1, 0.1, 0.2, 0.2], reg_class_agnostic=False, loss_cls=dict( type='CrossEntropyLoss', use_sigmoid=False, loss_weight=1.0), loss_bbox=dict( type='SmoothL1Loss', beta=1.0, loss_weight=1.0)):
      • PS: 在 BBoxHead 中的注释说明: 这个类表示的是最简单的 RoIHead, 只有两个分别用来进行 classification 以及 bbox regression 的 FC 层.
      • 1). with_avg_pool: 表示是否在进行 FC 前使用一个尺寸大小等于输入尺寸大小的 AvgPool 层先进行一次处理.
      • 2). with_cls: 表示是否包含 classification 用途的 fc 层
      • 3). with_reg: 表示是否包含 bbox regression 用途的 fc 层
      • 4). roi_feat_size: 表示输入进来的 roi feature map 的尺寸
      • 5). in_channels: 表示输入进来的 roi feature map 的通道数
      • 6). num_classes: 表示要分类的类别数目
      • 7). target_mean && target_stds: 进行 bbox_target 时候使用的参数, 是 bbox 解码过程需要的参数, 印象中没有错的话 这部分应该是加速训练收敛的
      • 8). reg_class_agnostic: 表示是否使用类别不可知的方式进行 bbox regression (也就是 bbox regression 分支输出的就是 4, 而不是 4 * num_classes)
      • 9). loss_cls: classification 的 loss config
      • 10). loss_reg: bbox regression 的 loss config
      • PS: 我们可以看到在 Faster RCNN 的 config 文件中, bbox_head 相关参数都是给 BBoxHead.__init__ 的, 基本没有给 ConvFCBBoxHead.__init__ 的参数, 应该是都靠其默认参数了, config 中只有 num_fcs 一个参数是给 SharedFCBBoxHead 的
      • 这个 __init__ 中主要进行的就是参数在类中的注册 以及简单的注册了 fc_cls 和 fc_reg 两个 FC 层
      • 接下来看 ConvFCBBoxHead.__init__
    • (3). ConvFCBBoxHead.__init__(self, num_shared_convs=0, num_shared_fcs=0, num_cls_convs=0, num_cls_fcs=0, num_reg_convs=0, num_reg_fcs=0, conv_out_channels=256, fc_out_channels=1024, conv_cfg=None, norm_cfg=None, *args, **kwargs):
      • 1). num_shared_convs: 表示 shared_conv 层的数目
      • 2). num_shared_fcs: 表示 shared_fcs 层的数目
      • 3). num_cls_convs: 表示 cls_convs 层的数目
      • 4). num_cls_fcs: 表示 cls_fcs 层数目
      • 5). num_reg_convs: 表示 reg_convs 层的数目
      • 6). num_reg_fcs: 表示 reg_fcs 层的数目
      • 7). conv_out_channels: 表示流程中 conv 层的输出通道数
      • 8). fc_out_channels: 表示流程中 fc 层的输出维度
      • 9). conv_cfg: 关于卷积层的配置
      • 10). norm_cfg: 关于 norm 的配置 (PS: 因为 Faster RCNN 中并没有使用 conv, 无论是 shared_conv 还是分支中的 conv, 所以 norm 也是木有配置的)
      • 流程: 除了进行参数值类内的注册之外, 还是会通过 _add_conv_fc_branch 进行各个阶段(指的就是 shared_conv, shared_fc 等这一系列的流程) 然后最后会在层后面接上一个 cls_fc_output 一个 reg_fc_output em… 接下来简单分析一下 _add_conv_fc_branch 函数吧
    • (4). _add_conv_fc_branch(self, num_branch_convs, num_branch_fcs, in_channels, is_shared=False):
      • 1). num_branch_convs: 表示 conv 层的数目
      • 2). num_branch_fcs: 表示 fc 层的数目
      • 3). in_channel: 表示输入的维度
      • 4). is_shared: 表示是否是前面 shared layers 的 flag
      • 流程: 就是按照参数去决定添加一下各个层次
  3. 接下来就对 forward 函数进行分析: 没啥好分析的, 就是按照流程各个模块走一遍最后得到 cls_score 以及 bbox_pred
  4. 之后会通过 bbox_head.get_target 获取进行损失函数的 GT 信息, 这个就要追溯到 BBoxHead 这个类上了, 有函数定义: get_target(self, sampling_results, gt_bboxes, gt_labels, rcnn_train_cfg):
    • (1). sampling_results: 上一个大阶段得到的 sample 出来的 proposals 结果
    • (2). gt_bboxes: GT BBox 信息 (实际上用不到)
    • (3). gt_labels: GT Labels 信息 (实际上用不到, 这两者用不到的实际原因应该是因为在 sampling_result 中已经将这些信息包含了)
    • (4). rcnn_train_cfg: config 文件中的 train_cfg.rcnn
    • 具体流程:
    • (1). 直接从 sampling_result 中提取出来 pos_proposals, neg_proposals, pos_gt_bboxes, pos_gt_labels, 然后将这些参数直接 feed 给 bbox_target 中
    • (2). bbox_target 会通过 multi_apply 调用 bbox_target_single 对每个 stage 的信息进行分别的 target_single 处理 (因为虽然上面的 features 已经是多尺度的信息放在一起了, 但是 sampling_results 还是分尺度的, 所以要用 multi_apply)
      • 分析 bbox_target_single: 主要目的就是将 pos 和 neg 的信息整合在一起, label_weights 表示这个标签应该占据的权重, neg_weight 是 1, 如果没有在 config 中具体设置一个 >0 的权重, 那么 pos_weight 也是 1. 同时只有 pos_sample 才会使用 bbox 信息.
      • 处理之后得到 labels, label_weights, bbox_targets, bbox_weights 然后会返回给 bbox_target, 然后在其中经过汇总拼接将拼接好的多尺度的信息返回
  5. 我们已经拿到了 optimize target 了, 接下来就要去拿 loss 了, 通过 bbox_head.loss 按照配置文件进行 loss 的计算配置 然后返回 loss
  6. 我们拿到了 loss 之后, 在 two_stage.py 的 forward_train 中返回这些 loss 即可
