Simple Summary for SNIPER: Efficient Multi-Scale Training

Posted on 2019-11-06

简单总结《SNIPER: Efficient Multi-Scale Training》

Introduction:

  1. 本文提出的SNIPER是进行instance level visual recognition任务多尺度的高效训练的算法. 并不去进行image pyramid所有像素的处理, SNIPER只处理GT instance在合适的scale上周围的context的信息. 说bg sampling 的 context-regions都是通过一个短周期训练的RPN网络提取出来的, 因此每个image在训练中生成的chips的数目都是根据场景复杂度自适应的改变的. SNIPER在一个单尺度(8001333)上仅仅需要处理30%左右的pixels即可, 而且其可以对极端大的resolution的图像进行处理(其实太大了导致物体超过了512512肯定是不行的), 提到了SNIPER针对重新采样的low resolution chips(512*512)进行操作 -> 因此可以在GPU上采取一个很大的batch(说因此就可以直接享受到batch-normalization的好处, 而不用跨GPU的去进行synchronizing batch-normalization). 作者认为, SNIPER将原图像划分为一个个的小的chips使得instance level recognition任务(主要指detection)更加倾向于image classification. 同时认为传统普遍接受的观点: 在high resolution图片上进行visual recognition十分重要可能并不是正确的(…其实你不也只是从大的图片上截取下来的吗…).
  2. 又是从人眼角度开始描述的, 同样认为人们只会关注一小部分的图片而周围的区域将会是模糊的, 所以是难以同时处理多尺度的. 通常是仅仅处理一小部分的image, 然后自适应的忽略掉图像剩余部分的视觉信息, 而主流的detector为了robust的scale representation而多尺度的去进行训练, 并且其会对所有的pixel进行训练. 同时认为多尺度会因为放大且全处理导致时间的大量增长, 在很多情形下是不可行的.
  3. 提到了之前自己的SNIP, 说在训练的过程中忽略掉特定scale下extreme resolution objects的梯度是有利于训练的. -> 也就是在特定尺度上关注特定大小的物体 -> 一个符合直觉的想法就是, 我们是不是可以不用将整张图片放大, 而只是sample在小物体周围的一个小的RoI chip就可以.同时从另一个角度说明这个选择就是, 当一个图像已经在一个high resolution了, 并且其中有很大的obj了, 那么upsample这些大的obj可能并不会带来好处(个人认为反而会因为感受野并不够大而错误判断一些事情)
  4. 作者接下来又说, 确实放弃掉image中很大一部分然后在upsample有助于节省计算量, 但是一个小的chip可能会因为缺少contexual信息去进行识别. 很大一部分的背景信息可能也会被忽略掉 -> trade-off between computation, context, negative mining. 也就是本文提出的训练算法(SNIPER), 其根据image content去自适应的从image pyramid不同尺度上去sample一些chips出来.
  5. 会根据GT instances sample一些positive samples出来, 根据RPN网络的proposal sample一些negative samples出来.
  6. 主要贡献: (1). 保证和SNIP相似性能的同时 可以减少3倍的处理pixel的数量. (2). 因为SNIP patch是比较小的, 所以可以享受到大的batch-normalization带来的好处(单卡20, 8卡160的batch-size, 即使使用的是resNet-101这种大backbone)
  7. 提到了关于R-CNN与Fast-RCNN的一个对比, (本文主要是通过截取固定大小避免了很多没有意义的物体的放大与缩小这一角度出发, 但是一篇知乎上的解读认为可以看做是对RCNN具有尺度不变性和F-RCNN的没有的分析, 虽然文章没有这么表述, 个人也并不认为本文是从这个角度出发的)(deformable conv2从另外一个角度进行这个事情的解释). 提到了F-RCNN会对所有image中的proposal进行upsample与downsample. 和RCNN不同, RCNN每个proposal都会因为SS算法而固定到224*224的大小, large obj与small obj都没有进行upsample与downsample. 而是只处理当前尺度下包含合适resolution的物体的区域, 认为这是比较合适的(文章中的角度主要是指没有将大物体放大, 小物体缩小这种不恰当的处理, 但是实际上这么做在de-conv2中解释是有利于尺度不变性), 但是Fast-RCNN与Faster-RCNN中已经明确分析了, 因为SS没有共享feature map以及SS算法是基于CPU进行的所以速度很慢. 本文认为提出的SNIPER算法享受了两者的好处, (1). 生成尺度相关的固定大小的chips, 并且使chips中包含尽可能多的合适scale的物体 (2). 在一个chips内使用Fast RCNN的做法, 去进行chip中多proposal的处理. -> 以上两点带来了好处就是, 不需要在包含大物体的地方进行upsample, 同时不用进行简单bg region的反复处理了. 相比于Fast-RCNN显著要快.
  8. 在SNIP的基础上, 对SNIP的 480, 800以及多个从1400 crop出1000大小的image pyramid进行处理, 去sample 512大小的patch. 实验发现, 在比较大(3*)的image上, 计算量可能仅仅是原先的1/10. 但是遇到的一个问题是, 当仅仅使用GT instance去生成patch的时候, 因为仅仅处理图像的部分内容, 导致训练中很少会对背景信息进行学习处理, 所以导致了假正率的很高, 因此后来还是要对bg进行negative patch的sample(sample可能包含false positive的region)(使用一个轻量级短周期训练的RPN去生成这些区域).

Method:

本文包含了两个模块: (1). positive/negative chip生成 (2). chip生成后的label assignment

  1. Chip generation:
    • (1). 首先SNIPER会在多个尺度{s1, s2 …}上进行chip的生成, 每个尺度上, 首先iamge会resize到 $W_i H_i$ 上, 然后 K K 大小的chips将会没 d pixels(本文是d=32)的进行生成(其实就相当于一个大的anchor… )
  2. Positive Chip Selection:
    • (1). 在每个scale上, 指定了一定的area range, 决定了每个scale上参加训练的GT boxes/proposals. 这些GT bbox构成了一个集合, 然后我们贪心的去选择能够将cover valid GT box最多数量的patch, 只有当一个GT box被完全的覆盖在chip中的时候才被称之为covered. 一个scale中的每个picture的所有的positive chip将会被combine 在一起.
    • (2). 上面的贪心保证了每个GT box都至少被一个patch所包含, 但是可能会存在一个GT bbox被同一个scale上的多个chip所包含, 也可能存在在不同的scales上被不同的chips所包含.
    • (3). 没有被完全被chip包含的GT box是被crop的, 但是cropped GT box仍然会保留在chip中, 并用来进行label assignment(???什么意思???)
    • (4). 至此, 所有的GT box都会在合适的scale上被covered. 同时提到了 因为crop-size是远小于图片resolution的, 因此SNIPER并没有在high resolution上对很多的背景信息进行处理. 这在处理high resolution image的时候大幅度的减少了计算量与内存需求.
  3. Negative Chip Selection:
    • (1). 提到尽管positive chips覆盖了所有的positive instances, 但是很大一部分比例的bg并没有被覆盖到, 因此导致了在训练的过程中缺少背景的训练, 导致了在inference的时候false positive的上升. 提到当前点的object detection算法中, 通常multi-scale training是经常被使用的, 在所有的scale上图片的每个pixels都会被训练, 这可以有效的减少false positive, 但是却带来了大量的计算损耗, 并不像这么做(因为这么做就发不了paper了 2333).
    • (2). 作者假设很大一部分比例的bg是很容易被区分的(算是常识… imbalance问题), 所以作者认为可以避免去使用这些bg region去进行训练.
    • (3). 为了去找到这些区域, 作者认为最简单的方法就是使用object proposals去找出obj可能存在的区域, 剩下的区域就是被认为很简单就可以区分的区域, 这些区域就是我们应该放弃的.
    • (4). 实现上面的方法, 首先用一些epoch去先训练一个RPN, 不适用negative chips去进行训练, 因为这个网络的目的就是帮助我们粗略的找到可能存在物体的regions(文章说的是可能包含false positive的区域, 其实是一个意思), 这个RPN不需要非常精准.
    • (5). 对negative chip的选择时, 首先不考虑那些很简单就能区分的区域, 然后排除哪些与positive chips cover的proposal. 之后对于每一个scale, 我们类似选择贪心的去进行选择negative chip, 每个chip都只要要包含M个proposals, 这个时候就得到了最后的negative chips set
  4. Label Assignment:
    PS: 需要注意的是, 进入训练的是每个patch, 而不是original input image pyramid.
    (1). 强调了本文是end2end训练的, 每个chips都会类似于Faster-RCNN去进行训练.
    (2). 关于RPN的训练, 不进行尺度的区分(所以当一个大物体被一个chip crop的时候所剩下的小的region仍然是在这个阶段valid的), 但是后续进行proposal处理的时候只有落在当前尺度下的GT box才会被用来训练, 其他的将会被忽略掉. 剩下的设定和Fast-RCNN相似.
  5. Benefits: 大部分点都是之前提到过的, 另外其提及到本作可以一定程度的表明, 图片中有相当一部分的context对检测是没有什么作用的, 但是也有可能是因为深度网络还不够深, 其感受野(经验感受野)不足以覆盖全图的所有pixels.
  6. Inference过程: 使用图像金字塔: (480, 512), (800, 1280), (1400, 2000)的image pyramid(括号中的数字描述的是范围信息), 然后谈到了训练和推断期间有效的区域是和SNIP相似的. combine detections 使用的是soft-NMS
  7. 关于为什么对那个筛选可能存在物体的区域的RPN只使用 positive sampling 进行训练 -> 作者进行实验发现, 当RPN没有进行negative sampling的时候 召回率是没有下降的(与mAP相反), 因为召回率并不考虑假正率, 所以可以认为只使用positive sampling进行训练的时候, 已经可以发现所有的positive regions. 个人认为, 我们仅仅使用positive sampling可以加快收敛可以大大减少这个RPN的训练时间.

Simple Summary for An Analysis of Scale Invariance in Object Detection – SNIP

Posted on 2019-11-05

简单总结《An Analysis of Scale Invariance in Object Detection – SNIP》

Introduction:

  1. 针对的问题是解决极端尺度变化下的物体识别与检测问题. 提到了通过使用各种不偷设置的input data对detector进行训练来比较其scale specific与scale invariant的设计, 本文通过使用小物体的分类去验证ImageNet训练的不同网络架构, 表明了CNN并没有足够鲁棒的尺度变化适应能力
  2. 基于上面的观点, 本文提出在图像金字塔相同的scale下去对detector进行训练与测试. 因为小物体和大物体很难被在小的scale和大的scale中被recognize, 所以提出本文提出了一个新颖的训练策略 – Scale Normalization for Image Pyramids(SNIP) 其选择性的反向传播不同尺度的object instances的梯度到指定的image scale.
  3. 从数据集的角度对为什么COCO数据集上验证的效果难以达到ImageNet上的效果 – COCO中包含的小物体相比于ImageNet太多了, 有超过一半的物体的尺度和ImageNet中的最小的5%大小相似, 90%以上的物体没有达到ImageNet中位数的尺度. 同时COCO数据级中小物体和大物体的差异性也十分的大, 小物体和大物体的差距可能在20倍以上 -> 因此对Network尺度不变性的要求十分的高.
  4. 同时认为从ImageNet的分类任务到COCO的检测任务的预训练也会因为两者物体scale的差异过大而导致domain-shift十分严重.
  5. 提到了一些为了解决尺度变化性与小物体检测的问题的研究: 比如(1). 将 浅层与深层结合去检测小物体, (2). 空洞卷积/可形变卷积去增加检测大物体的感受野, 在不同resolution上使用不同的预测去捕捉不同scale的object instance, (3). context的利用, (4). 多尺度训练, (5). 多尺度inference
  6. 提到了上面的研究对obj detection是很有帮助的, 但是还是有问题没有解决:
    • (1). 是否upsample image对获取足够好的object detection检测性能有所帮助, 为什么要将数据集的480640转变为 8001200. 是否可以在预训练ImageNet分类网络的时候使用低分辨率与小的stride, 然后将这个预训练CNN作为detection的backbone去进行fine-tune
    • (2). 当对pre-trained iamge classification网络在detector中fine-tune的时候, 是否object instances resolution应该被限制到6464~256256的范围中, 以及经过改变input image的大小之后, 是否所有resolution的obj都应该参与训练.
  7. 本文结构:
    • (1). Section 3: 通过在ImageNet分类问题的网络中输入不同尺度的input去检验网络的尺度变化性的性能. 同时对分类问题进行了多scale的设计(通过稍微改变CNN的结构) -> 表明了upsample对小物体来说是必要的.
    • (2). Section 5: 分析尺度变化性在obj detection中的效果, 训练和比较了scale-specific和scale-invariant两种detector设计. -> scale-specific指的是对每种scale使用不同的detector, 提到了, scale-specific中训练物体尺度的不变性以及和pre-trained分类网络中物体scale相似的特性能够帮助降低domain shift. 但是scale-specific设计会因为数据量的减少而造成性能的退化. 而训练单个检测器将会是学习任务相当的困难. (PS: 这么看SSD什么的就都是就是scale-specific得了? 因为在每个feature map中会单独的进行该层的学习.)
    • (3). Section 6: 本文的主要内容: 提出的新的训练策略, 其通过对在训练过程中减少scale-variation获得性能的提升, 同时其不用过多的关注因为训练sample减少所带来的惩罚. Scale-invariace 通过image-pyramid实现, 其将 每个 scale 的 obj 的 input representations进行normalize(每层只解决一个指定的尺度). 同时为了减小域迁移的问题(detection与classification)在指定的scale level上只针对特定resolution的RoI/acnhor进行反向传播, 采用上述的约束之后, SNIP可以利用所有的obj instances去进行训练 (个人认为意思应该是: 每个尺度都可以利用所有的obj instances去进行训练, 应该类似于之前看到的一篇人脸检测的augument方法: 将instance放缩到该scale上的合适的resolution范围之内, image也会放大该instance放大的比例, 然后将这个image作为input data输入进行训练) 提到了SNIP是可插拔的, 只要涉及到尺度的变化性都是可以使用的
  8. 认为的确特征金字塔形式的特征融合是有利于低层次访问到高层次语义信息的, 但是认为当物体的尺度过于小的时候, 即使进行了up-sampling(比如2的)得到的物体也是非常小的(远小于预训练分类网络input的224224) 因此, 高层次的语义特征不会对小物体的分类产生帮助(语义gap导致的), 同样大物体也是这样的 (PS: 看到这里思路和finding tiny face很像啊… 都是想往imageNet预训练的中尺度去靠拢, 而不是相办法去解决掉这个语义gap) 主要观点就是: 特征金字塔确实可以高效的利用network不同层次的features, 但是不能代替image pyramid去检测很小的/很大的objects.
  9. 提到了不能使用类似finding tiny face中的scale-specific detector的原因是因为数据量的问题(因为物体类别、姿态等比较多, 每个类别的不同尺度/姿态的数据量并不够), 作者对R-FCN添加了scale-specific filter之后 发现反而会破坏掉filter的性能.

Image Classification at Multiple Scales

PS: 这个部分作者研究了“当在训练和测试时不同resolution的images被当做input提供”时的domain shift产生的效果. 为什么这么做是因为, SOTA的detector通常使用8001200的resolution进行训练, 但是会使用image pyramid进行inference(且通常会用14002000的图片去检测小物体)

  1. Naive Multi-Scale Inference:
    • PS: 使用正常数据集图片downsample然后upsample的image进行inference, 使用正常尺寸的训练数据进行训练.
    • (1). 首先作者通过对original ImageNet database的图片进行downsample得到一组不同resolution的图片 4848 - 128128. 然后这些image会被再次的upsample到224*224并且作为输入去给使用原图片的CNN进行inference(CNN-B)
    • (2). 实验结果发现: 当inference输入图像的尺寸和训练使用的尺寸差距越大, 性能退化越明显.
  2. Resolution Specific Classifiers:
    • PS: 使用小image进行训练, 然后使用小物体进行检测(backbone net是修改过的)
    • (1). 基于上面实验的观察, 简单的提升小物体检测性能的方法就是去在ImageNet上使用不同的stride进行预训练. 提及到相比CIFAR10(包含小物体的数据集)的最优模型来说, ImageNet分类网络的最优模型在第一个卷积层直接选择使用步长为2(77)同时后面更上一个步长为2的pooling layer, 将图像中大部分关于小物体的signal抹去了. 因此, 对ResNet101的第一层选择使用步长为1, 33的conv去对4848的image进行处理, 使用55conv以及步长为2进行96*96图片的处理.
    • (2). 可以发现这么做的效果相比于在训练和推理中使用不同的尺度要好多了, 所以认为可以对low-resolution的物体使用一个单独的object detection去对low resolution进行训练.
  3. Fine-tuning High-Resolution Classifiers:
    • (1). 提出了另外一种针对小物体检测的简单的解决方式: 对上面 Resolution Specific Classifiers的inference以及train的input进行改变, 进行一个upsample(然后仍然保持原有的分类backbone的结构) 可以发现效果相比于 2 做法更好了.
    • (2). 认为进一步表明了对小物体使用high-resolution image进行学习是很有必要的.
  4. 得出的结论是: 当训练obj detector的时候, 我们可以选择使用不同的网络架构去对不同resolution的obj进行分类, 也可以选用一个单独的架构去进行所有resolution的分类. 因为ImageNet的pre-training是有益的, 且基于大obj instance学习的filter是有助于对小物体分类的. Upsampling images并且使用基于high resolution的image预训练的网络应该可以比专用的网络更好的分类小物体. (提到了 确实现在的网络通常都会使用同一个架构+upsample而不是多种针对不同尺度的网络去进行检测 -> 符合本文的分析)

Method:

  1. 提到了本文使用的baseline是: Deformable-RFCN, 提到了D-RFCN是在RFCN基础上在conv5层天界deformable convolution去自适应的改变网络的感受野去生成不同scale物体的scale invariant的特征. 同时进行最后的roi pooling的过程也是使用Positive Sensitive RoI Pooling. 后面的实验中使用什么没太看明白… 是值得是RFCN部分和RPN部分使用的是不同的backbone进行训练的吗? 好像为了减少训练时间与GPUmemory所以这么做的.
  2. Data Variation or Correct Scale? :
    • (1). section3(讨论3中对classification训练与推断策略的实验) 表明了training与testing阶段resolutions的不同通常会导致性能的明显下降. 提到了使用比较小的图片进行训练的原因之一是因为GPU显存的限制, 所以对于训练来说是要比测试的时候resolution要小的. b本 section对image resolution, object instance scale, data variation对object detector 的性能进行了一定的分析. 使用不同的设置进行训练, 但是会同样的使用14002000分辨率的图像去进行小物体的检测(COCO数据集中指的是小于3232像素的instance).
    • (2). (Chapter15--Exercise) Simple Summary for Dropout: A Simple Way to Prevent Neural Networks from Overfitting

Posted on 2019-11-02

Chapter15–Exercise 简单总结《Dropout: A Simple Way to Prevent Neural Networks from Overfitting》

PS: 文章30页实际上对前馈神经网络的主要内容在前7页… 以思想为主, 后面的就先不看了

Abstract & Introduction:

  1. 出发点: 强调了Overfitting是在深度学习中很常见且很严重的一个问题, 并且因为大型的深度网络很慢, 所以我们不可能通过combine多个大型神经网络的方式去在test time中使用.
  2. Key idea: 训练过程中在network中随机的drop掉一些units及与其相连的connections. -> 认为这可以有效的避免units之间co-adapting过度.
  3. 在训练过程中, 会从指数级别的可能的”thinned” network中进行sample出一个网络进行训练, 在测试的时候, 认为简单的使用没有经过 “瘦身” 的网络就可以是一个对所有瘦身网络的效果的近似.
  4. 强调了因为deep neural network强大的学习能力, 能够对非常复杂的输入到输出的关系进行关系的建立. 但是因为训练数据是有限的, 所以 很多的复杂关联的建立可能是对sample中噪声的关系建立, 因此会导致overfitting. (包括(1). 在验证集中性能下降的时候就停止训练 (2). 引入L1,L2正则化等对权重的惩罚, (3). 权重共享(soft weight sharing) 在内的一系列方法都想要解决这个问题)
  5. 认为当计算能力无限的时候, 最好的对固定尺寸的model进行regularize的方法就是将所有可能的parameters的设定得到的prediction结果进行综合考量, 根据给定的训练数据的后验概率进行加权 -> 认为需要的计算太多了, 本文提出的dropout正是一种意义上等同的指数级的共享参数的模型的weighted grometric mean.
  6. 强调模型的组合确实总能提升性能, 但是大型network中是不可行的 成本过大的. 同时说将明显不同的模型机型组合是最有帮助的, 这些模型需要结构与训练数据都不同, 然而训练很多个模型结构是很困难的(因为要找每个模型的最优解的超参数, 且数据通常也是很有限的), 而且这么做也会导致inference时候的缓慢.
  7. dropout指的是在network中隐藏但是可见. 即暂时去掉其所有的输入与输出连接(将它孤立^_^) 选择drop掉的unit是随机的(提到了关于保留比例: 0.5的比例对中间层大概是不错的, 对输入单元接近1是比较好的)
  8. unit的状态有保持和dropout两种状态, 所以相当于一个有N个unit的dropout网络是 $2^N$ 个网络的汇集, 但是因为权重的共享, 还是 O(n^2) 个参数. 本质上可以看做相当于每个网络都只训练很少的次数甚至没有进行训练
  9. inference阶段采取近似的方式而不是穷举所有可能网络进行加权, 就是之前提到的使用一个网络进行输出, 简单的调整就是一个unit保留的概率是p, 那么对其输出权重用p进行加权
  10. 文章还提到了不仅仅对前向网络有效, 对其他的图模型也是有效的, 比如Boltzmann Machines.

Motivation:

  1. em… 说是从基因得到的灵感… 染色体会从parents双方各获得一半的基因, 然后添加很小可能的随机突变就产生了一个后代. 所以认为这种从parents获取信息仅仅做微小改变的对优化个体的很好的方式, 因为一组很好的”genes”可以通过直接传递给offspring而很好的共同work. 认为sexual reproduction(有性生殖)从另一个角度看就是 打破基因之间的 co-adapted, 认为因为co-adapted 可能会因为已经形成了一个比较复杂的co-adapted系统而阻碍适应性. 因此认为这是一种最优的机体进化出来的方式.
  2. em… 越说越玄学… 认为这种方式的优越性的解释可能是因为自然界的长期选择可能不是根据个体的适应能力, 而是根据基因的混合性. —> 有性生殖不仅仅是让好的新的基因进行广泛的传播, 而且是通过降低co-adaptations去使新的gene对现有系统有一定的提升空间, 而不是因为过度的co-adaptation而阻止新gene产生的效果.
  3. 因此认为dropout会强制要求hidden unit必须学习更加鲁棒的能够提升自己表达能力的features 而不是依赖于其他的hidden unit去纠正其自身的错误.

  2. 同样可以将dropout看作为一种随机的正则化技巧

Model Description & Learning Dropout Nets

  1. 整个基本概念的描述就是在layer中加入一个阶段去进行一个服从伯努利分布(其实是二项分布)的参数r的采样, 然后和layer的原输出进行相乘, 用这个值代替原来的输出值. (PS:因为前馈的时候输入没有向后传递, 所以反馈的时候也并不会更新, 所以不需要显示的将前面的连接删除->因为一定是0)
  2. Backpropagation: 可以使用SGD方法可以进行, 但是是在thinnet上进行的propogation(both forward and backward). 每个parameter会在mini-batch中进行平均.(提到这一点应该是因为不是所有的parameter都会被更新.) 为了强调方法的普遍适用性, 强调了填的类似momentum, 衰减学习率, L2权重衰减等在dropout基础上设置后仍然都是有效的.
  3. 强调了 强制使 新的每层的weight vector的norm值小于一个固定的值c是一个对dropout非常有效的regularization.
  4. 给出了一个组合: dropout + max-norm regularization + large decaying learning rates + high momentum. (max-norm正则化作用之一是防止因为过大的学习率导致参数超出一定的范围, 而使学习到的参数更新到一个很不好的值)
  5. Unsupervised Pretraining: 对预训练网络同样进行dropout, 提到了最初担心dropout会导致对网络中原有的信息造成破坏. 但是提到了 最终实验表明当学习率与训练随机初始化网络的最佳学习率一致的时候会有这样的问题(应该指的就是性能退化), 当时采用更小的学习率的时候会获得更好的性能

Introduction:

  1. backbone网络对detector的性能很重要. 本文是使用现有的backbone通过构建一个更加强大的backbone(比如ResNet和ResNeXt)去达到一个更加的detection performance.
  2. 主要思路是将多个backbone实体通过一个有效的策略去assemble到一起去形成一个more powerful backbone, 文章将这个称为(Composite Backbone Network: CBNet).
  3. CBNet会迭代式的将前一个backbone的输出feature feed给下一个backbone, 前面阶段的features被称为high-level features. 这么级联的feed之后最后一级的feature map将会被用来进行object detection.
  4. CBNet可以很容易的集成到现有的SOTA的detectors中(因为本质就是换了一个backbone啊… 而且也不是多尺度的输出), 并且可以显著的提高性能. 同时实验表明CBNet对分割问题也可以有很大的提升.

  6. CBNet的前几个backbone都叫做assistant backbone, 最后一个叫做Lead Backbone.
  7. 将high-level到low-level的流动看做了features的融合, 认为因此能够提升detection 的性能.
  8. 本文贡献总结: (1). assemble多个相同的backbone为一个更强的backbone, 且可以轻易的集成到现有的SOTA的detector中 (2). 对目标检测和分割都是一个很好的backbone.

  1. Architecture of CBNet

    • (2). 在传统的单backbone中 $ x^l = F^l(x^{l-1}) $ , 但是在CBNet中显而易见的就是 $ x_k^l = F_k^l(x_k^{l-1}+g(x_{k-1}^l)), l >= 2 $ g(·)就是两个layer进行组合的映射了. 文章中g使用的是两个1*1 conv layer(用来减少channel数量的)跟上一个batch normalization layer, 之后进行upsample操作.

    • (3). Dense Higher-Level Composition(DHLC): 将前面的backbone当前stage的后续的所有输出层都进行模块的处理, 然后和后面的backbone的对应层进行fusion (PS: 不加个权吗?… 明显应该前面level的feature map的权重高一点有利吧…)
  3. CBNet的detection network部分: …… 无所谓, 只是提供一个backbone…

简单总结《Objects as Points》(CenterNet)

PS: 这篇文章是大家所说的CenterNet, 和另外一篇名字叫做CenterNet的是同期的作品, 但是思路不太一样, 这篇文章真的是可以快又比较准, 又可以实时的达到很好的效果, 并且其可以在多领域都取得很好的效果, 很NB.

PS: 其实对VOC数据集的效果没有想象中好.

Intruduction:

  1. 提到Detection通常用axis-aligned box去确定objects, 大部分obj detector枚举所有obj可能在的位置然后进行分类, 认为这样是低效而浪费的.
  2. 本文提出了一种其他的方式 -> 将obj看做一个single point(bbox的中心点), detector通过keypoint estimation去找到中心点然后去进行其他object properties的回归(比如2D检测的size, 3D检测的location, orientation, 人体pose等)
  3. CenterNet是轻量、快速高效且准确的, 并且作者在多个领域的数据集上进行了实验都有着很好的效果
  4. 主要观点就是sliding window based object detector因为枚举所有的可能的object location与dimensions所以是很浪费的 -> 预测物体中心
  5. 主要做法: input输入 -> 经过fully conv network进行处理 -> 得到heatmap -> heatmap中的peak points认为是obj的中心 -> 对每个peak point的Image feature进行objects bounding box的height和weight的predict. 模型用标准的dense supervised learning进行训练, 推断的时候并不进行non-maximal去进行后续的处理
  6. 预测中心点之后 1). 对2D obj detection: 预测width 和 height 2). 对3D obj detection: 预测obj absolute depth, 3D bbox dimension, obj orientation 3). 对human pose: 预测每个关节位置相对于中心节点的offset

    • (3). 训练方法是按照CornerNet提出的训练你方法进行训练的, 对每个GT keypoint p, 先将其映射到低分辨的feature map上(即p’=floor(p/R)), 然后将所有的gt keypoint按照这个方法贴成一个GT heatmap, 在贴的过程中以每个点做一个高斯分布每个高斯分布的方差是物体尺度自适应的参数(CornerNet中的设定是1/3的radius, radius在CornerNet中设定的是保证corners产生的anchor与GT bbox的overlap大于等于0.7的那个radius).

简单总结《CenterNet: Keypoint Triplets for Object Detection》

PS: 这个centerNet好像不是大家说的那篇, 公认的centerNet应该是这个->Objects as Points

Introduction:

  1. 背景: keypoint-based方法通常会导致很多的不正确的object bounding boxes, 认为是缺少向cropped region中查看所导致的, 本文则提出额一种有效的解决方式, 用很少的代价就能探索cropped region中的信息辅助detection.
  2. 是基于CornerNet进行的改进
  3. CenterNet使用三个点去定位物体, 能够提升精度与召回率. (左上 右下 中间三个点)
  4. 本文设计了两个模块: (1). Cascaded corner pooling (2). center pooling 去进行两个corner的信息的丰富以及提供central region的有效信息->去确定是不是个obj 减少1中所提到的不正确的obj bbox的情况的出现
  5. 认为cornerNet的问题还是在于没有充分的考虑全局信息去确定物体, 而是通过边界的检测以及点的feature embedding去确定物体, 也就是其对边界的检测比较敏感, 但是对哪两个角点进行组合并没有很好的去进行利用. -> 因此导致了误检的情况的大量出现, 作者认为, 只需要考虑到额外的信息就可以很好的解决误检问题.
  6. 本位就是基于角点为其提供额外的信息, 使角点组合之后能够通过中心正确的判断这个bbox对应的是不是一个正确的obj. 选用中心区域作为辅助的额外信息的intuition是: 如果一个预测的IoU和GT有着很高的IoU, 那么在其central region的中心keypoint有着很高的概率被预测成同一个类别, 反之亦然. -> 也就是在推断的时候 根据corner生成了”anchor”之后, 通过判断其中心区域是否有一个同类别的关键点来进行是不是该真实物体的判断.
  7. 大致介绍(1).center pooling: 用来预测center keypoints的一个分支. 用来帮助center keypoints获得足够多的物体辨别信息 从而使得更加容易去关注(perceive)一个proposal的中心部分.(通过获取水平方向和垂直方向上的最大点进行该操作) (2).cascade corner pooling: 在原有的corner pooling的基础上使其拥有感知接受内部信息的能力.(We achieve this by getting out the max summed response in both boundary and internal directions of objects on a feature map for predicting corners.)

Method:

  1. Baseline and Motivation
    • (1). 使用CornerNet作为baseline, 重新梳理了一下CornerNet的思路(注意一下, left-top和right-bottom heatmap中分别选取top-k个点作为角点去进行匹配. 一个最后输出的bbox的confidence score是两个角点置信度的平均值)
    • (2)作者对CornerNet进行了一个实验, 其在MS-COCO验证集上对CornerNet的FD(false discovery)进行了计数, 说实验表明了, incorrect bounding boxes占据了很大一部分的错误, 即使当IoU threshold很低的时候(PS: 这里指的IoU threshold很低指的是被判断为是incorrect的bbox与GT的overlap很低的意思, 不是指训练或者nms的那个IoU threshold), 当IoU threshold设置为0.05的时候有32.7%的错误, 设置为0.25的时候有36.8%, 0.5的时候有43.8% (PS:这可以说明生成低IoU的错误判断的bbox占据了大部分所生成的bbox) 对小物体来说就更是如此了.(个人认为小物体本身就很难去与其overlap, 所以相对来说比较多, 而且小物体也很难通过看内部去进行是否是该物体的判断(因为可能小物体只占据feature map上的一个pixel, 容错性就已经很差了))
    • (3). 提到了为了能够是CornerNet看到bbox内部的信息, 一种方法是使用two-stage的detector, 使用RoI pooling去看向bbox内部的visual patterns. 但是又觉得two-stage方法太慢了.
  2. Object Detection as Keypoint Triplets
    • (1). 总体的框架还是没什么变化的, 仍然是采用了cornerNet的结构, 产生heatmap, 预测embedding以及offset, 选择top-k的corner以及匹配策略去生成bbox(一看到这里就知道这个不是大家普遍说的cornerNet了…) 但是细节上还是有变化的
    • (2). 细节上:
      • 1). 网络结构上除了corner分支还添加了center分支, 但是center分支并不进行embedding的预测, 只是生成heatmap以及offset, 生成heatmap经过了本文所谓的提出的center pooling, 同时corner分支上将corner pooling替换成了cascade corner pooling
      • 2). 在预测了offset并且组成了bbox之后 人为的定义一个central region, 然后去检查在central region中是否包含这个类别的center keypoints, 如果检测到了就保留这个anchor, 如果没有检测到就不保留
      • 3). 最后bbox的置信度分数更改为 左上角点、右下角点、中心点的置信度的平均值
    • (3). central region的大小定义很明显将会影响到最终的检测结果. (eg: 小的central region将会使得small bboxrecall rate变低, 大的central region将会使得large bounding box的precision变低), 作者因此选择scale-aware central region去自适应的根据bbox的size调整central region的大小.(即小bbox产生大central region, 大bbox产生小central region)
    • (4). 因此得到的region区域的坐标是与n相关的, n越小占比越大, n越大占比越小, 文章是设置大于150anchor的n=5, 否则n=3
  3. Enriching Center and Corner Information
    • (1). Center pooling: 认为物体的几何中心并不一定要表达出非常有辨识度的visual pattern(eg: 认为比如对人来说, 人头才是有辨识度的, 但是center point通常在人身体上) -> 基于这个问题提出了center pooling 来捕捉you辨识度的视觉信息. 具体做法是: backbone输出feature map之后, 去决定是否pixel在feature map中是一个center keypoint, 找水平方向和垂直方向的最大值, 然后把他们加起来. -> 认为可以有效的帮助辨别center keypoints (PS: 为啥不像corner pooling一样从右侧和下侧/左侧和上侧取, 而不是整行取 -> 个人认为是因为center的确定需要四个方向的关键值)
    • (2). Cascade corner pooling: 认为corner经常是在obj外面的, 缺少local appearance features, 所以认为Corner pooling的目的实际上是为了寻找边界方向上的最大值来去决定角点. 但是, 这么做使得角点对边过于敏感. -> 希望能够让角点看到obj内部的visual pattern. 因此提出了Cascade Corner pooling

    • (2). 511的input size -> heatmap size为128*128 使用Adam作为优化器
    • (3). Loss组成: 1). 对角点和中心点都进行focal loss(针对置信度), 2). 对corner进行pull loss和push loss的计算(是针对embedding的, 想要同一对角点的embedding相近, 不同对的原理, 具体细节请看cornerNet论文), 3). 以及对corner和center计算l1-loss(针对位置)

Simple Summary for Grid R-CNN Plus: Faster and Better

Posted on 2019-10-28

简单总结《Grid R-CNN Plus: Faster and Better》

PS: 是对Grid R-CNN的改进, 作者都没有变化, 但是这篇文章把代码开源了, 文章也很短只有4页, 主要讲述改进的方面.
PS: 部分描述来源于 知乎这篇文章

Introduction:

  1. 出发点: Grid R-CNN虽然精度高, 但是相比于faster rcnn要慢.
  2. 问题1: Grid R-CNN对所有的grid-points都使用相同的representation region去生成supervision maps(就是N*N个grid point feature map使用同一个完整的feature map去生成heatmap), 认为这是很低效的(eg: top-left grid point不应该在right或者bottom的区域出现) -> Grid R-CNN Plus将会只使用1/4的最可能的区域去进行supervision. -> 从而减小grid branch上的feature map的size, 同时认为能够集中关注区域而带来grid location准确性上的提升.
  3. 同样, 文章对sampling strategy, normalization method, NMS stategy, hyper-parameters进行了一定的分析 -> 使得模型能够和faster rcnn速度相当.

Method:

  1. Grid Point Specific Representation Region
    • 因为只有positive sample(IoU > 0.5)才会被选择送给Grid branch, 所以在supervision map(GT点)上只可能出现在一个有限的区域中, 所以我们不应该使用全部大小的feature map去进行每个grid point的feature map的提取, 这么做是十分没有效率的, 同时也不是很稳定. -> 5656变为2828 只用可能的区域去进行提取 并且这么做作者视为一个normalization… 因为其原来分布是不normal distribution的 而在局部是normal distribution的
  2. Light Grid Head:
    • 由于最后的输出尺度降低了一半，所以也可以同时将grid branch中的其他特征图分辨率也降低，eg:14x14->7x7。前面的RPN+ROI Align产生一个固定的feature map 14x14，接着使用一个步长为2的3x3卷积核，然后再使用7个步长为1的3x3卷积核从而产生7x7分辨率的特征图。紧接着我们将这个特征分成N组（默认为9）,每一组关联一个grid point，接着使用两个组反卷积将特征图尺度变为28x28，强调了group deconvolution可以加速上采样的过程。另外一个好处是，由于我们对每个grid point的表达进行了归一化，因此他们变得更加closer,导致在特征融合时不需要使用很多的卷积层来覆盖这个间隙。在Plus版本，只使用了一个5x5 depth-wise卷积层来代替原来的3个连续的卷积层。
  3. Image-across Sampling Strategy:
    • 提到了因为grid branch只是用了positive proposal去训练, 导致了在不同sample batch中positive samples的数量上的不平衡. (eg: 一些图片上可能有很多个positive samples, 然而其他的可能仅仅有几个positive samples) 这种情况可能会导致grid branch上的feature distribution不稳定.
    • 因此选用了image-across的sampling策略, positive proposals会几张图片中进行sample, 当一个image中sample出来了少量的positive proposals的时候, 会从其他的包含多的positive proposal的图片进行batch的构成. 因此不再规定在一个图片上sample多少, 而是限制两张图片sample数量
    • PS: 个人认为一可以减少单张图片的positive-negative imbalance问题, 同时batch之间的positive也是可以相对保证的
  4. Non-Maximum Suppression Only Once
    • 原来的Grid RCNN需要两次NMS，第一次是proposal的生成，只选择Top 125个样本进行边框矫正，第二次是做最后的classification，作者发现，尽管只是一小部分的proposal，进行80类的NMS还是很慢，所以在Plus版本，直接移除了第二个NMS，同时将第一个NMS的IOU阈值设置为0.3，分类阈值设置为0.03，只选择100个proposal进行进一步的分类和回归。

PS: 个人认为值得强调的一点是: 每个grid point的feature map虽然变小了, 但是仍然是使用整个RoI Align的feature map得到的, 只是经过处理后得到的对应grid point的feature map变小了. 这个feature map是对应着缩小范围了的可能的分布的.

  6. 本文的主要贡献:
    • (1). 提出了Grid R-CNN, 其能够通过全卷积的有效保留空间信息的network代替传统的基于anchor回归的network
    • (2). 设计了一个multi-point supervision去进行grid中的points的预测, 从而减少不准确点带来的影响. 而且本文进一步的移除了feature map层次的information fusion机制, 使得spatially related grid points能够获取聚集周边grid point的特征信息, 从而能够更加准确的校准点的位置
    • (3). 进行实验, 表明了其具有很好的效果
  7. 提到了最近的工作: R-FCN使用region-based的全卷机网络替代原来的包含FC的网络(其目的是解决location的不敏感性问题), FPN进行feature fusion融合了多尺度的feature map, Mask R-CNN从增加精度(roi align)以及联合分割的多任务loss出发.
  8. LocNet提出了boundary-based方法以进行object detection的准确定位(通过物体边的概率分布去计算物体的位置), 其依赖于region boundaries的条件概率, 而本文的方法是基于grid points prediction的. 同时认为LocNet是用来进行proposal generation的, 而Grid R-CNN是用来bbox prediction的.
  9. CornerNet: 具体思路见总结, 认为其是bottom-up的方法, 也就是直接从整张图片去生成关键点而不定义像anchor之类的实体. 认为cornernet的一个关键部分就是keypoints的识别与正确分组. 而本文方法则是top-down的, 先在第一个阶段定义instance, 关注的是怎么准确的定位grid point. 后面又重复了一遍贡献

Method:

  1. 总体结构框架: 前部分仍然是Faster RCNN的标准结构, 将RoI替换为RoIAlign之后, 不再像原来一样进行bbox的regression, 而是使用grid guided机制去进行定位(PS: 但是分类还是分类的), grid prediction使用的是一个全卷积结构, 输出heatmap(文章中说采用的是FCN结构), 通过这个heatmap我们可以定位与obj对准的bbox的grid points. 然后通过这些grid point, 我们最终就能够从feature map层次的information fusion中判断决定obj bbox的准确性.
  2. Grid Guided Localization:
    • (1). 提到了自己全卷积进行object localization的方式: 设计了一个NN的grid, 每个grid point的feature map都与proposal的bbox对齐, 文章以一个33为例子就是传统的9个点的选择位置. 也就是按照R-FCN的思路, 针对不同的位置设计不同的feature map(在这里叫做heatmap可能更好), 仍然通过mask rcnn的roi align生成固定的1414大小的从原始图像中提取出来的feature map, 然后经过8个33的空洞卷积(说是为了更大的感受野), 之后接上两个反卷积去进行放大, 将这 NN 个feature maps放大到5656的大小, 然后跟上一个sigmoid函数, 每个heatmap都进行这个操作得到了各个grid point的概率图(目标区域5*5的cross shape 都被看做是positive 的location -> 应该是为了减少训练难度以及鲁棒性)
    • (2). 之后在推断阶段会从每个heatmap中选取最高的confidence的pixel, 并且使用原图像相应的位置作为一个grid point.
    • (3). 然后我们会利用这些点去决定bbox的boundary. B=(x_l, y_u, x_r, y_b), 每个值都通过在这条边上的点的位置乘以其概率分布进行加权求和去进行确定(PS: 那中间点的点的意义何在?)
    • PS: 这就是之前所说的不再像cornerNet一样仅仅根据两个点去确定bbox, 而选用更加稳定的方式进行的做法了
  3. Grid Poitns Feature Fusion:
    • (1). 认为grid points之间是有内部关联的, 其位置可以通过其他的grid points去进行校准以减少总体上的偏差, 因此作者提出了一个空间信息融合module
    • (2). 认为最直觉上的做法就是进行坐标的average, 但是feature map中的丰富信息就因此被丢弃了. 所以作者认为进一步可以来提取grid points的local features去进行进一步的fusion操作. 但是仍然认为会丢失不同feature map 响应位置的信息. eg: 以3*3的grid为例, 对左上角如果只是用左上角的feature map的左上角region信息可能会丢失掉比如中间点的feature map的左上角region所能够带来的信息贡献. 因此, 作者希望能够设计feature map level的information fusion机制去完全的利用信息
    • (3). 对NN个grid point的feature map 使用不同的filter去提取信息, 可以得到新的NN个feature map
    • (4). 对每个grid point, 邻近的grid point(L1距离为1的grid point)都会在fusion中产生贡献, 然后这个过程也需要进行卷积, 每个j gridpoint对i gridpoint产生贡献前需要进行三个5*5卷积(说是进行information transfer). 然后相关的grid point的feature map将会fuse一下产生新的feature map.
    • (5). 之后还会再加上一轮feature fusion操作, 并且参数也不是共享的, 这一轮操作认为可以汇聚L1距离为2的gride point的信息, 认为是更加可靠的
  4. Extended Region Mapping
    • (1). Grid prediction module输出的具有固定空间尺度的heatmap代表了grid points的分布置信度. 的那还是可能存在GT grid point的位置并不在roi中, 所以natural idea是放大proposal area.

Simple Summary for Cascade R-CNN: Delving into High Quality Object Detection

Posted on 2019-10-27

简单总结《Cascade R-CNN: Delving into High Quality Object Detection》

Introduction:

  1. 提到文章的初衷: 我们现在设定一个IoU阈值去进行positive和negative的区分定义, 但是一个不是很高的阈值通常会导致noisy detection的产生, 然而过高的IoU threshold则通常会导致性能的退化, 认为原因有两点
    • (1). 训练的过程中因为正样本vanishing positive samples导致的过拟合
    • (2). inference阶段训练好的最佳模型得到的IoU与input hypothesis的不匹配(???啥意思…)
  2. 本文Cascade RCNN的architecture包含了一系列的detectors, 使用逐渐增长的IoU thresholds进行训练, 来逐步的减少假正(false positive)情况的出现.
  3. 本文detector是stage-by-stage的训练的, 认为是基于一个detector的输出对于下一个更高质量的detector来说是一个很好的分布.
  4. 认为obj detection是由(1).distinguish foreground objects from background (2).assign object to proper obj class labels (3).localization obj三个问题组成的, 认为这三个问题都很难解决, 因为detector存在着很多的false positive问题, detector必须要找到true positive的同时对false positive进行抑制
  5. 认为原本的通常的做法(设置threshold为0.5)不够好, 因为会放过很多接近于true positive的false positive
  6. 强调了一下本文定义的quality是 与GT的IoU大小, quality of detector是 采用训练的IoU threshould.
  7. Basic idea: 一个单独的detector只能对一个特定的quality level达到最优.
  8. PS: 作者示例图的意思是用 RPN得到的IoU作为location performance, regression后的out作为detection performance. 每个bbox regressor在IoU接近训练的threshould的时候是有最好的检测性能的, detector也是训练的时候选择什么阈值检测最后的结果就是什么阈值最好的. -> 基于两个图的实验, 作者认为选择一个threshold进行训练是不能得到多种情况下的最优值的.
  9. 同时认为直接选择一个高的threshold进行训练是反而会降低性能的, 认为原因是, proposal detector所假设的分布通常会因为imbalance问题倾向与低IoU的, 选择大的IoU就意味着只关注那较小数量的样本(就是1.(1).提到的问题). 以及认为high threshold只对high quality (distribution) hypothese上成立, 在很多没有见到过的实际input中只是次优的.(就是1.(2).提到的问题).

Method:

  1. 提到了之前的认为RPN阶段不够的文章:
    • (1). iterative bbox: 认为存在两个问题: 1). 只采用了一个threshold, 其对于更高的IoU的对象来说是次优的, 最后造成了对很高的IoU的对象的性能的退化 2).bbox的distribution随着RPN阶段的变化之后分布发生了明显的变化, 对第一阶段RPN最优的设置对于后来的阶段可能是suboptimal的. 从而导致了其需要进行很多的人工设置, 并且通常情况下简单的重复两遍并不会带来性能的提升.
  2. Detection Quality:
    • (1). 提到了一个自然而然希望能够将多种IoU threshold聚集在一起的设置方法: 用一组不同IoU threshold的loss去进行, 但是实际上是不可行的, 因为可以发现在原始阶段IoU边高的时候样本数量是急剧下降的, 所以导致不同threshold的loss其positive样本数量上存在一定的问题
  3. Cascaded Bounding Box Regression: 其实和iterative BBox结构上没有什么区别, 但是实际上是有区别的: (1). 认为iter bbox是迭代的refine bbox的过程, 而自己是想要通过重采样改变bbox的分布的过程 (2). Cascade BBOX在训练和推理阶段都使用, 所以认为在训练和推理两个阶段之间的分布式不存在差异的… (3). 认为iter bbox每个阶段使用同样的处理”函数”, 而自己是不同的(threshold改变了) (PS: 除了(3)前两个像是凑数)
  4. Cascaded Detection: 又列举了一个实验图, 强调RPN proposal所选取的bbox是低质量的(低IoU的), 因此会影响到后续对高质量anchor的训练过程, 本文方法通过cascaded bbox regrssion将roi分布重采样到有利于高质量anchor训练的程度上.
    • 每个阶段都会进行classify以及regressor, 损失函数也是这么加权的

PS: 个人认为: 为什么都说anchor set的初始设置会影响和GT的overlap导致一定问题的存在但是还是设置anchor set -> 还是因为classification之后的信息仍然是位置信息不敏感的, 需要设定一个范围值去进行预先设置, 然后进行回归. 如果直接在原位置上进行center点的预测可能会导致效果的不好. PS: 可能这就是为什么cornerNet等选择使用Hourglass作为backbone而不是使用resnet做.

Introduction:

  1. 本文提出了一个Recurrent Rolling Convolution(RRC)的结构使得->deep in context
  2. 强调定位精确度(localization quality), 本文通过循环的引入context到bounding box regression, 强调high IoU threshold条件下
  3. 还是老观点: 绝大多数的低质量的bbox是因为小物体和overlap物体的错误定位导致的

Method:

  1. 首先强调了faster-rcnn等two-stage方法使用last layer feature map相比于原图像因为经过了几个pool的处理变得很小. 对小物体检测造成了很大的影响. 而使用multi-scale输入图片是一种解决办法, 但是相对来说效率会很低
  2. 认为在SSD中强调的观点(low resolution->big obj, high resolution->small obj)是很有道理的
  3. 但是说SSD性能是一直比不上two-stage方法的, 同时当对IoU要求越来越高的时候, 两者之间的差距越来越大. 认为原因是: 检测结果基于了一个太强的假设 -> 每一层feature map都要独立的去对其scale所产生的output负责 -> 每一层feature map都需要有足够的信息去支持detection以及object的精确定位. 即:
    • (1). feature map需要有足够的resolution去表示物体的细节信息
    • (2). 将image转变为feature map的映射需要足够深使得信息能够在feature map上达到一个足够抽象化的语义信息
    • (3). feature map需要包含适当的contextual信息
    • 强调可以发现, 在上一层和下一层相比, 深层次反而resolution要小, 而浅层次的深度不够, 认为相对来说难以训练
    • 认为Faster RCNN则不存在这个问题, 但是Faster RCNN仍然存在没有足够的resolution去表示物体的细节信息的这个问题
  4. 基于3中所陈述的原因, 本文提出的network本质就是尝试将各个阶段信息进行融合, 具体式子和形式见paper吧
  5. RNN for Conditional Feature Aggregation

    • 认为中间层次要能同时追溯到浅层和深层(也就是feature fusion, 本文用concate完成的)
