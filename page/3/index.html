简单总结《Enhancement of SSD by concatenating feature maps for object detection》

  1. 提到了添加feature map可以提升效果, 但简单的仅仅添加feature map有瓶颈, 本文考虑如何高效的利用feature map去提升性能
  2. 通过改变network的结构, 改变的是classifier backbone后面的部分
  3. 本文考虑到了两点:
    • (1). 将不同层的feature map进行联合考虑
    • (2). 增加feature pyramid中layers的channel数
  4. 本文实现方法:
    • (1). 仅仅是pooling: 除了通过卷积得到的feature map之外, 还将上一层的feature map进行一个pooling操作, 然后将二者concate到一起
    • (2). 仅仅是deconv: 同上面操作, 只不过后面会将低层进行反卷积操作然后到高层上进行concate
    • (3). pooling + deconv: 两种操作都要
    • PS: 细节说了, 说了pooling+deconv时候每个feature pyramid都是2816channel的, (512, 1024, 512, 256, 256, 256)concate的

Simple Summary for Context-Aware Single-Shot Detector(CSSD, DiCSSD, DeCSSD)

Posted on 2019-10-19

简单总结《Context-Aware Single-Shot Detector》(CSSD, DiCSSD, DeCSSD)

  1. 文章想要解决的目的是: SSD没有很好的利用proposal box之外的context
  2. 所谓的DiCSSD, DeCSSD分别指的是使用Dilated convolution和Deconvolution的CSSD
  3. 实际有效感受野实际上远远小于理论感受野, 所以提出我们需要更多的context去判断
  4. 两种CSSD实现方法都是先扩大ERF到一定的scale上, 然后将这些fuses到一起产生新的feature map 或者复用高层次的feature map去产生不同感受野的feature map
  5. 强调了感受野对小物体检测的作用(也引用了finding tiny faces那篇文章的观点)
  6. DeCSSD就是将高层的feature通过deconv以及后续接上conv后和resolution最大的那个feature map进行fusion然后fusion到一起形成一个新的feature map, 其余的高层feature map保持不变
  7. DiCSSD就是每层通过dilated conv将各层的feature map的感受野扩大, 然后层内进行weighted-sum fusion, 产生一系列的feature map去进行predict
    • 这个思想引用自Multi-scale context aggregation by dilated convolutions
  8. 后面的对感受野的描述先略过了… 有时间回头看
  9. PS: 文章用DeCSSD的时候感受野(context引入)的设计不知道是否考虑到了文章所分析的内容… 看看源码应该就知道了

Simple Summary for FSAF:Feature Selective Anchor-Free Module for Single-Shot Object Detection

Posted on 2019-10-18 Edited on 2019-10-23

重新总结《Feature Selective Anchor-Free Module for Single-Shot Object Detection》

一、文章思路

  1. 希望能够根据feature/context去对instances进行特定层次的指定以及分配, 而不是按照传统的启发式的认为浅层(小size)就是小物体, 深层(大size)是大物体

二、引用观点与具体细节

  1. 陈述当前传统anchor-based detector的问题
    • (1). 启发式的特征选择(确实很玄学)
    • (2). overlap方式的anchor sampling
  2. 提到了FSAF模块可以自己工作进行anchor-free的detection, 也可以和anchor-based method结合去达到更好的效果
  3. [Seeing Small Faces from Robust Anchor’s Perspective]
    • (1). 点击这里见总结
    • (2). 文章引用主要观点: 改进设计了针对小物体的anchor (文章主要思想是认为小物体不容易与anchor overlap, 所以导致了regression到GT的困难, 基于这个观点进行了一系列tricks的引入, 个人觉得前两个(1. 改变feature stride 2.增加非中心的anchor)比较靠谱)
  4. [Bounding box regression with uncertainty for accurate object detection.]
    • (1). 点击这里见总结
    • (2). 文章引用主要观点: 说将bounding box的分布视为高斯分布去更好的进行定位 (是利用对Loss的改进使得anchor更加准确同时在NMS中引入了一些信息的一篇文章)
  5. Ground-truth 和 loss的设计
    • (1). 对bbox划分了有效区域和忽略区域
    • (2). classification输出的是K个map, 每个instance根据bbox划分的有效区, 忽略区, 无效区域产生的贡献不同: 有效区域视为positive, 无效区视为negative, 忽略取则不进行梯度的回传. 同时相近尺度上的忽略区域(这个区域包含了有效区域)也不进行梯度回传
    • (3). regression输出是4个MAP, 分别代表四个偏移量(距离中心上、下、左、右的距离), 都是类别无关的 输出时除以一个常数, inference的时候再乘上…
    • (4). loss的使用方面: 对分类结果使用focal loss, 对box regression结果使用IoU loss.
  6. Online Feature Selection
    • (1). 根据给出的loss 算最小的那个scale作为训练的scale
    • (2). 为什么选取loss 最小的思想是… 认为最小的是当前feature loss的lower bound, 进行feature 参数的调整, 可以降低这个lower bound
    • (3). 认为在推理的时候, 我们不用选择feature level, 因为认为适合的层会输出最高的置信度

Simple Summary for Bounding Box Regression with Uncertainty for Accurate Object Detection

Posted on 2019-10-18

简单总结《Bounding Box Regression with Uncertainty for Accurate Object Detection》

PS: 大概思想明白了, 没有仔细看为什么…

  1. 文章的Motivation: 大规模数据集会倾向于将GT BBox定义的尽可能的clear, 但是很多情况下bbox的label是仍然模糊不清的 -> 模糊不清指的是: (1). 单纯的标注不准确 (2). 遮挡带来的模糊 -> 应该是因为label定义的不清楚/不是最优 导致了网络对物体边界的学习比较模糊, 所以导致了bbox在loss中占据了不应该占据的loss
  2. 本文提出一个boundingBox regression loss去同时学习bbox transformation与localization variance
  3. 认为传统的L1 Loss等并没有考虑到这种模糊的bbox label, 导致了有时候训练得到最高confident的anchor并不是最佳的anchor
  4. 作者先将bbox prediction和GT bbox看做高斯分布与狄拉克函数, 然后loss使用预测分布和GT分布的KL divergence作为新的bbox regression loss, 认为这么做的有点有
    • (1). 可以捕捉到dataset中的ambiguities, bbox regressor因此会从ambiguous bbox中获得更小的loss
    • (2). 学习到的variance在post-processing中也十分有用
    • (3). 学习到的概率分布式可解释的
  5. 文章想要预测每个位置的置信度, 用一个简单的Gaussian函数去进行定义, 就是以GT中心的高斯分布函数, σ标准差与置信度有关, 越小越确定, 当=0时就是GT了此时是狄拉克函数
  6. 损失函数计算 GT的狄拉克函数以及高斯分布的KL散度 然后作者将这个loss换了一个形式(具体就见文章吧)
  7. Variance Voting: 在NMS中引入了分布信息, 具体见论文吧… 前面就已经和要找的方向跑偏了… 有时间回来看

Simple Summary for Seeing Small Faces from Robust Anchor’s Perspective

Posted on 2019-10-17

简单总结《Seeing Small Faces from Robust Anchor’s Perspective》

PS: 阐述了一个很有意思的观点, 用一个我觉得不算严谨的EMO Score说明了为什么, 然后用一些方法去改进anchor. 个人觉得算是a bag of tricks吧 而且好像tricks都是旧的…

  1. 本文主要内容: (1). 设计了新的anchor设计想法去支持很大程度的尺度不平衡的人脸检测问题(尤其是小人脸) (2). 提出新的Expected Max Overlapping score -> 理论上解释low overlap问题(就是为什么小脸不容易被anchor overlap)并且能用一些有效的anchor design策略去达到更高的face overlaps
  2. 本文定义小脸->less than 16*16
  3. 因为发现, 当前的anchor设计不能保证小脸和anchor boxes的high overlap
  4. 1.(2). 中策略包括了
    • (1). anchor stride reduction with new network architectures
    • (2). extra shifted anchors
    • (3). stochastic face shifting
  5. 提到了anchor-based detectors在shape与appearance invariance上有好的表现, 但是在scale invariance上表现不是很好
  6. 前面提到的low overlap问题是当进行分类以及对anchor box进行bbox regression之后高confidence的box并不和tiny faceoverlap的问题
    • 在训练前对每个face计算overlap最高的IoU -> 然后将faces分到几个不同scale的组中 -> 在每个scale的组中计算平均最高IoU score
    • 得到结果: average IoU与recall rates成正相关 -> 也就是本来的box就和小脸overlap很小和大脸overlap大
    • 认为与small faces有小的overlap的anchor难以调整到GT上所以使得小脸的难以检测
  7. 提到了
    • (1). 减小anchor stride(看上去就不是一个feature map 对应pixel对应一组anchor了)
    • (2). 同时对anchor进行shifted, 而不是原本的中心以使得anchor变得更加密集化
    • (3). stochastically shift the faces in order to increase the chance of getting higher IoU overlaps(这是啥操作…)
    • (4). 用多个anchor去match low-overlappedfaces
  8. EMO scores
    • (1). 首先提及传统的anchor-based的设定:
      • 原先anchor被视为GT的原则: 当anchor与obj overlap超过了一定的threshold或者anchor是和该物体overlap最大的anchor
      • anchor stride指的就是 feature map 尺度相比于原图的stride
    • (2). 计算EMO Score:
      • 假设一张脸可以随机的出现在图片的任意位置处, 我们去这张脸可能存在的点为(x, y) 那么这张脸在全局出现的概率就可以看做为一个概率分布函数 p(x, y)
      • EMO Score = maxIoU*p(x, y)对x和y的二重积分
      • 我们一般假设脸是均匀概率出现在各个位置的, 所以积分就变成了对 maxIoU 在不同位置的积分, 这个值是与anchor set的设置等有关的
      • 因为每个anchor都是在空间上平移的, 所以对每个位置进行积分和对一个anchor stride大小的patch中积分是等价的
      • 同时因为anchor关于patch的中心对称, 所以我们只要关注 1/4 个patch即可
      • 计算方便这里假设用和某一个anchor大小设置相同的face进行计算得到最后的结果作为EMO Score
      • 可以看出Face Scale与anchor越大, stride越小EMO越大
      • PS: em… 设定大小和anchor一样大 em… 能概括大部分情况吗? 而且其设定的一组anchor值即使是和anchor重叠最小的时候都保证和这个anchor重合…
  9. Anchor Design的策略
    • (1). Stride Reduction with Enlarged Feature Maps:
      • 因为anchor stride和feature stride相同, 所以认为扩大feature map可以帮助, 三种upsample方式
        1. 双线性插值upsample
        1. 双线性插值+skip connection
        1. 空洞卷积代替down Sample
    • (2). Extra Shifted Anchors
      • 我们不去改变feature stride的大小, 我们改变anchor stride与feature stride的关系, 使得anchor stride < feature stride
      • 很简单的多添加anchor, 只是中心位置发生变化
    • (3). Face Shift Jittering
      • 对图像去进行shift -> 使得每个脸和anchor的overlap都可能增加
    • (4). Hard Face Compensation
      • 当一个脸和一个anchor大于一个threshold的时候会被当做positive, 当所有的anchor都小于这个阈值的时候会将几个anchor作为positive em…
  10. 空洞卷积效果很好

简单总结 《Adaptive Feeding: Achieving Fast and Accurate Detections by Adaptively Combining Object Detectors》

PS: 只看观点思路 没看细节

  1. 提出Adaptive Feeding 去将一个 Fast but less accurate detector 与一个 Accurate but slow detector 进行结合, 适应性的决定一个图片困难还是简单去决定使用哪一个detector进行分类
  2. 首先利用一个很简单的很快的detector首先生成一些instance proposals, 并基于这些内容去决定使用快的还是准的model去进行后面的步骤
  3. 优点
    • (1). AF快的同时准确
    • (2). 可以将两个不同架构的模型进行直接组合利用, 而不需要额外的training data
    • (3). AF使用一个不平衡的学习框架去从hard images中辨认easy ones(???), 并且这个框架可以调整speed和accuracy之间的trade-off
  4. hard 还是 easy 认为应该是由image中的instances决定影响的 -> 因此在AF第一阶段放一个 instance generator去提取决定 easy vs. hard 分类的特征

  6. 对于这个 easy vs. hard SVM分类器, 认为并不简单, 因为要考虑到不平衡以及cost-sensitive 问题 -> 因为大部分的images是简单的, 只需要都猜简单我们就能达到一个看上去很好的性能, 这显然是不行

Simple Summary for Finding Tiny Faces

Posted on 2019-10-15

简单总结 《Finding Tiny Faces》

PS: 知乎上的文章精读分析
PS: 说实话这篇文章观点很多… 但是重点不太清楚… 而且我又不熟悉人脸检测 记录的很多东西只是作为小物体检测观点的使用与积累吧 遇到实用的回来再看 总结暂时就这么简单

  1. 本文关注的寻找小的人脸的三个方面:
    • (1). 尺度不变性
    • (2). 图片分辨率
    • (3). context 推理
  2. 对不同尺度分别训练detector
  3. 表明context是很重要的, 并且使用很大的感受野
  4. 提及到很多工作都使用的是scale-normalized classifiers(比如滑动窗口以及利用ROI-pooled image features的classifier), 这种做法将regions resize到了一个canonical template size. 但是作者认为我们应该怎么去确定这个template的size使其能够检测到小物体的同时检测到大物体. 因此作者训练了多个针对不同尺度(以及aspect ratios)的detectors. 但是训练一大组的针对特定尺度的detector会有如下的问题
    • (1). 训练很多个detector会缺少对应尺度的训练数据
    • (2). 在推理阶段会导致低效
    • 解决上面问题: 以一个多任务的方式进行 特定尺度的detector的训练和运行，使用从不同featrue map层共同提取的新的feature map (应该也就是共享feature map的思路?) -> 但是说这么做提升了大目标的检测准确率但是对小目标仍然存在挑战
  5. 如何从pre-trained network中提取scale-invariant features: 说本文证明现在的network是 tuned for 特定的尺度。 为了能够将从这些network中提取的features扩展到新的尺度，使用如下的策略:
    • 在测试阶段(???)通过插值和抽取去对图片resize
  6. 提到很多识别系统都利用了图像金字塔(multi-resolution)的方式, 本文发现对最底层的pyramid进行插值对找到小物体十分有效
  7. 本文使用的方法: 用粗粒度的图像金字塔去处理目标的大尺度的变化，为了小物体的性能, 引入足够的context信息将所有template设置成统一感受野大小的多尺度模板(即小template会加入多一些的context信息)，在从多个feature map层中提取的feature进行template的定义以及获取
  8. 对于如何更好的对context进行encode: 一个大问题: 如何有效的对大规模的image regions进行encode -> 多尺度信息的融合
  9. 提到了大量提升context对成功识别定位小脸很有帮助，大脸并不需要这么做，因为有足够多的特征信息
  10. 强调了特征融合带来的好处
  11. 文章认为因为imageNet训练集基本都是中尺度目标以及采用的数据集也是大物体很少，小物体很多，所以任何分类器训练之后都会对中尺度检测效果更好->将大小物体都尽可能的靠对中尺度物体的检测去检测出来->从而引出使用图片金字塔(0.5->为了大物体, 1->为了中物体， *2->为了小物体)
  12. scale-specific detection: 在不同尺度的input image上进行多个template的训练(每个template使用 t(h, w, σ)表示在σresolution上检测 (h/σ, w/σ) 大小的物体)，然后选择性能最好的针对特定尺度的template -> (应该是这个操作解决推理慢的问题，并且说prune掉差的template之后反而会提升效果)
  13. 整个Architecture的pipeline: input image -> 创建(2, 1, *0.5)的image金字塔 -> 使用CNN去获取每个尺度的 feature map -> 使用训练、筛选、剪枝好的template去进行特定尺度下的检测

Simple Summary for A Unified Multi-scale Deep Convolutional Neural Network for Fast Object Detection

Posted on 2019-10-15

简单总结 《A Unified Multi-scale Deep Convolutional Neural Network for Fast Object Detection》

PS: MS-CNN
PS: 使用数据库: KITTI and Caltech

  1. 整个框架包含两部分: (1). proposal sub-network (2). detection sub-network
  2. proposal sub-network中，对多尺度的output layer进行detection(多尺度是为了匹配不同scale的obj)
  3. 针对不同尺度的detector将会结合去产生一个更强的多尺度obj detector
  4. feature upsampling 通过 deconv 进行 -> 说是减少内存以及计算量的消耗(因为不是对原图像这种大的东西进行放大了)
  5. 改进1: 单一尺度会因为感受野的统一而不能很好的匹配各种尺度的obj -> multi-scale 为了解决感受野大小和物体大小之间的不一致问题
  6. 改进2: 通过对 feature 进行 upsampling 而不是对 input image 进行 upsampling
  7. 陈述了几种覆盖匹配多种scale物体的方式 ((1).-(4).是针对非CNN的)
    • (1). 对 input image 进行多尺度的scale 然后用这些尺度的input image去得到多尺度的feature map，而只需要学习一个简单的分类器 (对原图像进行尺度放缩然后每个都进行特征提取虽然精度高，但是计算量消耗太大)
    • (2). 取而代之的训练多个分类器，每个分类器对对应的尺度的物体进行检测 (共享feature map)
    • (3). 对input image进行几次(很少)rescale, 同时学习一些model template(也就是分类器)
    • (4). 对input image进行几次(很少)rescale, 但是利用插值去补充哪些缺失的feature map
    • (5). RCNN简单的 warp proposal patches为CNN正常的自然尺度与(1)类似，但是不用计算整张图片的feature map，只需要特定的 patch 进行计算即可
    • (6). RPN: 共享了原图的feature map, 用RPN后的框利用前面共享的feature map进行操作, 类似(2)
    • (7). 但是认为RPN对不同尺度的物体使用了同样尺度的 model template可能会导致严重的scale inconsistency
    • (8). 本文做法: 在不同尺度上的feature map上使用几个不同size的template 进行处理 PS:其实就是在提取到最后一个conv的特征图之前的特征图上进行feature map的提取、匹配
  8. upsampling部分:
    • (1). 认为 input image upsampling 并不会丰富图片的细节
    • (2). 使用deconv的 feature rescale 代替 input image rescale

Simple Summary for Is Faster R-CNN Doing Well for Pedestrian Detection?

Posted on 2019-10-14 Edited on 2019-10-15

简单总结 《Is Faster R-CNN Doing Well for Pedestrian Detection?》

PS: 使用的数据集: Caltech, INRIA, ETH, and KITTI

PS: 关于hard mining的 cascaded boost forest 就先不看了… 不是关注的重点

  1. 发现RPN检测的性能可以达到一个单独的行人检测器的效果，但是后续的分类器的步骤反而使得性能下降了, 原因认为是:
    • (1). feature map的resolution不够去对小物体进行检测
    • (2). 缺少bootstrap机制去对 hard negative examples 进行mining
  2. 解决的方法是: 在RPN后面加上一个基于 共享的high-resolution convolutional feature maps的 boosted forests.
  3. 1.(1). 的解决说法是: 从更浅层但是更高分辨率的层pool feature进行feature的pool 并且使用空洞卷积(hole algorithm/`a trous)增加feature map 的size (这里的意思是在保持感受野大小不变的同时不减小feature map 的resolution)
  4. 1.(2). 问题: 行人检测问题中hard background instances(像是人的东西)是造成错误的很大的原因, 在通用物体检测中不太存在这个问题(主要错误原因是类别之间的错误，前景后景的错误其实也挺重要的) 本文使用 Cascaded Boosted Forest方法进行解决，并且不使用手动设置的特征去训练forests而是复用 RON 的深层卷积特征 -> 既通过复用减少了训练时间，同时利用了深度学习的特征
  5. 认为原先使用 conv5_3 层之后的信息会导致小物体的信息分辨率会小于 7*7 (设定的统一的输入到最后分类器特征大小)，会导致信息的没有辨别力. 而使用高层次的信息(也就是重新计算的高分辨率的feature map的信息会更加有辨别力)

细节:

  1. 因为行人的ratio大概是相等的，不想通用物体一样变化很大，使用一个aspect ratio即可，多个anchor是为了找到不同尺度的物体
  2. 文章并没有用空洞卷积进行RPN的fine-tune，而是单独的用作提取更高resolution的feature map (理解没有错的话是单独又做了一遍feature map的提取, 与RPN的那一条路是分离的两条路)

想法:

  1. 仔细思考一下是不是 在交通标志检测中也存在 hard negative examples
  2. 利用空洞卷积保证感受野的大小的同时保证较高的分辨率是不是一个可行的行为
