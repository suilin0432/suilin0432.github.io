Simple Summary for Deep Clustering for Unsupervised Learning of Visual Features

Posted on 2019-11-18

简单总结《Deep Clustering for Unsupervised Learning of Visual Features》

PS: Unsupervised 方法(Self-Supervised), 在综述贴中, 描述这篇文章描述的主要是 利用对feature representation的clustering去进行parameter的更新

PS: 确实本文的方法是无监督的方法, 通过聚类的方式产生了 伪标签去进行训练来得到好的representations, 但是最后的结果是怎么产生的??? 聚类类别明显是很多的啊, 最后是怎么对应到各个类别的呢? 而且怎么用来去做分割已经detection也不是很明白… 是后面要进行有监督的学习进行fine-tune吗? 那使用VGG什么的带来的效果看上去并没有那么好.

Introduction:

PS: 总体流程上个人认为还是整个pipeline, 在feature提取层上取出来对应的提取的特征, 用来进行无监督的分类, 然后给上一个伪标签, 后续进行到分类网络的最后进行使用这个伪标签的训练去反向传播(图并没有画最后面的一部分)

  1. 想要将clustering以一种end2end的方式融合到深度学习网络中, 并且是其能够基于大规模的数据集进行训练. 本文提出的是DeepCluster, 一个联合学习神经网络参数和对得到的features结果进行聚类标签的聚类算法. DeepCluster迭代式的使用标准聚类算法(K-means)对features进行聚类, 并使用subsequent assignments作为监督去进行network的权重的更新. 使用DeepCluster去无监督的对CNN在大数据集上进行训练. 最后可以达到SOTA效果.
  2. 从Fig 1中可以看出的本文的主要做法: 将没有标签的数据进行clustering做分类, 之后形成伪标签, 然后用伪标签去进行训练.
  3. 提到了传统的聚类算法通常是针对具有固定feature的线性模型进行的设计, 这些工作也很少能在学习features的同时很好的工作.
  4. 本文, 作者提出了新颖的聚类算法, 针对大规模的convnets的end2end的训练. 表明了模型能够使用聚类网络获取到有用的通用features. 本文的做法就是: 交替的进行(1). 对图像描述子进行分类 和(2).通过预测cluster assignments进行convnet的权重的更新. 文章说是为了简单才选择使用的Kmeans, 其他聚类算法也是可行的. 认为与self-supervised方法不同的是, clustering只需要很少的domain knowledge, 并且不需要inputs的特定signal.
  5. 本文同样通过修改实验规则证明了框架的robustness(修改了包括convnet网络结构以及数据集), 也强调了不仅仅能捕捉class-level information, 同时也能捕捉instance-level information.
  6. 主要贡献点: (1). 一个新颖的使用如k-means的基础聚类算法的end2end训练的unsupervised方法, 同时仅仅需要很少的额外步骤. (2). 在很多的unsupervised transfer任务上SOTA的性能. (3). 一个关于unsupervised feature learning的验证标准的讨论.

Method:

  1. 首先对一些流程、目标与定义进行了公式化的描述
  2. Unsupervised learning by clustering:
    • 首先强调了, 当对提取特征的network参数进行随机初始化的时候, 并不会产生好的features, 但是random features的性能要比瞎猜要好多了. 认为这个好的效果是由于conv本身的结构所带来的对input signal的强先验导致的. 本文的idea就是去利用weak signal去促进convnet的辨别能力.
    • 本文对convnet的output进行cluster, 然后使用后续的cluster分配的伪标签进行最后损失函数的更新, 这个过程会迭代的进行去进行features的学习然后进行分类
    • 这里指的convnet应该是没有接上后面进一步辨别网络的部分, 只是用来学习特征的, 文章使用k-means作为基本的聚类算法(但是作者观点是按照以往的工作, 这里使用什么聚类算法都是无所谓的)
    • k-means使用的分类标准是几何距离
    • 引入了一个 d * k (d是feature的维度, k是分类的类别)的centroid matrix C, 通过 $min||(f_{θ}(x_n))-Cy_n||^2_2$ 其中$y_n$的每一个entry都是0/1,且只能有一个1. 对这个问题进行优化来得到最佳的伪标签y_n以及矩阵C(虽然后续C没有使用)
    • 总体上就是, $min||(f_{θ}(x_n))-Cy_n||^2_2$ 去得到伪标签, 然后去进行分类任务的损失函数的更新进行网络参数的优化.
    • 但是提到了这么操作很可能会产生平凡解, 需要一定的操作去避免优秀解的退化
  3. Avoiding trivial solutions
    • 强调了平凡解(应该指的是所有的点都被分到了一个类别, 或者大部分的点在少数的几个cluster中的时候)的而存在, 不仅仅是对非监督训练neural network来说的, 对于任意联合学习discriminative classifier和labels的方法都存在平凡解的问题. 提到了discriminative clustering即使在线性模型中使用的时候也会出现这个问题. 提到了解决办法通常是: 约束或者惩罚每个cluster中的点的数量, 但是提到了因为涉及到了整个dataset中的计算处理, 不适合用于大规模数据集上的处理办法, 所以提出了一系列其他的想法:
    • (1). Empty clusters: 提到了出现所有点在一个类别的可能性之一在于没有限制empty cluster的产生. 提到了一个最常用的方法就是自动的去重新分配empty cluster -> 当一个cluster是空的时候随机选择一个非空cluster并且将这个非空cluster的中心添加一个小的随机扰动之后作为空集的新的中心, 然后重新的去将非空集合按照这两个中心点进行一次聚类
    • (2). Trivial parametrization: 提到了当分类时候将大部分的image分配到了少部分的cluster中时, 学习到的parameter θ会对这几个类别非常有辨别能力. 尤其是当极端情况下只有一个类别的时候, 这个时候分类类别将会百分百正确. 这种情况通常在数据类别不平衡的时候出现. 避免这个问题出现的一般策略就是从uniform distribution中进行image的sample, 或者使用伪标签.

Introduction:

PS: 对tractable和scalar进行一下解释: (1). tractable是可解的意思, 指的是定义的P(model)通过最大化data likelihood去求解P(model) intractable就是不定义P(model) 但是可以sample出来data (2). scalable是可扩展的意思, 指的是模型在input size变大的时候复杂度的增长是仍然是可以接受的

  1. 强调了对natural image的分布进行建模是无监督学习中很重要的一个问题. 这个任务需要image model能够同时是具有表现力的, 可解的且可扩展的. 本文提出的深度模型是在二维空间维度上一个像素一个像素的去进行image的生成预测. 本文方法的能够对raw pixel values进行discrete probability进行model, 并且将image中的全部依赖关系进行encode???
  2. 模型结构的创新点在于: (1). fast two-dimensional recurrent layers (2). 在deep RNN中添加了residual cnnections.
  3. 强调了当前模型的一个难点是在于怎么使得生成模型是tractable且scalable的(可解&可扩展). 提到了很多工作都是基于随机因变量模型(eg: VAE)去提取meaningful representations, 但是通常面临着其具有一个intractable inference step的问题, 阻碍了其性能. 另外对于tractable model来说通常会考虑 将图像中的pixels的joint distribution看做是条件分布的乘积 -> 将问题变成了一个序列问题, 即模型会根据所有已经生成的pixels去进行下一个pixel的生成. 但是认为这样的模型(复杂度)都是高度非线性的, 需要考虑到长程关系, 因此在high resolution上是低效的. 综上所述, 文章描述了现有的模型都是对可解和可扩展(其实意思是当input规模增长的时候)
  4. 本文在RNN(SLTM)模块中引入residual connection的原因就是希望能够加深网络 -> 可以堆叠12层.
  5. 本文提出了两种LSTM单元: Row LSTM, Diagonal BiLSTM. 同时考虑对模型进行简化得到 PixelCNN, 发现CNN也可以去用来构建一个具有固定依赖范围的序列模型(通过masked convolution).
  6. 本文对pixel value的预测使用的是离散的值(通过一个softmax层实现的multinomial distribution), 而不是通常大家所做的连续的值. 并且实验证明对本文提出的模型可以提升最后的性能.

Method:

  1. 强调了一下, 本文的目标是估计一个natural image的分布, 以用来可解的计算image的分布概率, 并能够生成新的image.
  2. Generating an Image Pixel by Pixel:
    • (1). 目标就是对每个n*n的图片x生成一个概率分布p(x). 可以把image写成一个序列, x_1, …, x_n^2. 为了预测联合分布, 会把其写成条件概率分布的成绩 p(x) = ∏p(xi|x1,…,x_i-1)
    • (2). 因此p(x_i|x_1, …x_i-1)就是预测当前像素值的分布函数了, 每个pixel生成的顺序就是从左上开始一行一行进行的
    • (3). 因为图像是3通道的, 所以每个位置的值是由3个值所确定的, $p(x_i|x_{<i}) = p(x_{i,R}|x_{<i})p(x_{i,G}|x_{<i},x_{i,R})p(x_{i,B}|x_{i,R},x_{i,G})$ (也就是三个像素值之间也是有着条件概率关系的).
    • (4). 提到了在进行训练和验证的时候每个像素的值的计算都是并行的, 但是在进行图像生成的时候是sequential的去进行生成的
  3. Pixels as Discrete Variables:
    • 之前的方法对image中pixel值的预测是连续分布, 本文则把这个考虑成了离散分布(通过把上面(3)中提到的公式的每一个p项考虑成一个用softmax实现的multinomial)把值划分为256个离散的值就可以了, 认为这么做形式简单同时不再需要对其distribution的shape做特定的prior分布, 同时文章实验发现, 使用离散的分布式能产生一个更好的结果的.
  4. Pixel Recurrent Neural Networks:
    • (1). Row LSTM:
      • 强调了Row LSTM是单向的layer, 从上到下的一行一行去计算feautres. 计算是使用一维的1 * k 的 convolution进行的, 对一个像素x来说, 这个层次会对其上的一个倒三角型的区域进行计算(因为每个点会考虑几个像素, 然后上一层的点又会考虑再上一层的几个像素, 一层层就拓展到成了倒三角) k越大对每层捕捉的context越多. 认为共享的权重有利于保证translation invariance.
      • 整体流程是: 先对当前阶段的输入x_i用一个卷积进行处理, 对上一阶段的hidden state h_i-1进行一个卷积处理, 两者维度都是4h n n, 将两者相加, 然后用sigmoid激活函数(针对output、forget、input三个门), tanh函数(这怒地content gate这个门)处理一下的到了四个门的参数. 然后用四个门的参数、前一阶段的hidden state与前一阶段的cell state最后可以求出当前阶段的hidden state以及cell state.
      • 提到了, LSTM层中都会有 input-to-stage 以及 stage-to-stage 两个component, 这两个component共同决定了LSTM中的四个门. 为了提高计算的并行性, 对于input-to-state component首先会进行全图的input map的计算, conv是使用带有淹没的只考虑有效context的conv实现的, 最后得到的tensor的尺度是4h n n, 代表了input map上的每个位置的gate vectors. 之后再得到了前一阶段的 hidden-state 以及 cell-state 之后就可以计算当前阶段的两个值了.
      • 强调了针对对input-to-state component进行加权的conv(masked conv)是被提前计算好的(文章后面应该是有介绍的) -> LSTM的一次一次的迭代我们就可以得到了最后的state值了.
      • 提到了因为Row LSTM的receptive field的问题, 感受野是不可能去捕捉到整个全图的范围的
    • (2). DIagonal BiLSTM
      • 希望不像Row LSTM一样捕捉的是倒三角区域的感受野, 而是希望能够利用全图的context. 从两个角点开始, 每个角点都对其下方以及(右侧/左侧)提供context(中间点只会对下方提供context)
      • 提到了使用大的一维卷积都不会改变考虑的context的范围是全局的这一特点, 所以使用更大的kernel是没有很大帮助的. (…但是明明这个是local的…)
      • 具体操作就不看了
    • (3). Residual Connections: PixelRNN会训练到12层那么深 为了提高收敛速度以及解决反向传播的问题(爆炸、消失)所以使用残差连接. 就是每层简单的加一个shortcut
    • (4). Mask Convolution: 提到了每个pixel的h channel的features是会被分为3个部分的, 分别对应于RGB的通道. 当在预测R通道的时候, 只有上面和左面的pixel才能被作为context, 当预测G通道的时候当前的pixel的R通道也会额外被当做context, B通道同理. 为此提出了mask conv这个东西, 有两种mask的方式: 1). Mask A: 其只对PixelRNN中的第一个conv layer中使用, 也就对应着前面所说的, 只有相邻的pixel(这个应该根据文章不同的两个LSTM模块的设置去进行这个相邻的定义, Row针对的是上面的3个像素, DIagonal指的同样是按照其定义的相邻)的值以及当前pixel的自己通道之前的通道的值会产生作用.(实现方法是保证每次更新之后都将不应该放入连接的位置的值设置为0) 2). Mask B: Mask B会在除了input连接的第一个conv之外使用, 将连接放开为不限制一定要是预测完的channel才会进行连接.
  5. PixelCNN: 提到了Row LSTM和Diagonal BiLSTM layer度有一个潜在的无边界的依赖范围, 从而使得每个state的更新需要sequential的更新, 带来了额外的计算量. 认为一个直接额想法就是让RF变大, 但是不是没有边界的. 所以希望能够使用传统的CNN去进行边界的限定, 直接生成所有位置的features的计算. PixelCNN使用多个masked CNN layer去进行, mask仍然是为了防止看到future context. (从描述上来看个人是认为, 多个local的CNN的使用就不用每次都要从头考虑到全局的信息了, 但是生成的时候仍然是需要一个一个的生成的, 仍然是不能并行的)
  6. Multi-Scale PixelRNN: 多尺度PixelRNN是由一个unconditional PixelRNN和一个或者多个conditional PixelRNN组成的. unconditional PixelRNN首先用一个小一点的s s的image去进行生成, 然后conditional network将小的image作为输入去生成一个更大一点的n n的image

Simple Summary for Semi-Supervised Learning with Ladder Networks

Posted on 2019-11-12

简单总结《Semi-Supervised Learning with Ladder Networks》

Introduction:

  1. 本文将supervised learning与unsupervised learning在深度网络中进行结合. 提出的model被训练去同时减小supervised与unsupervised cost function. PS: 本文一个贡献就是对LadderNet进行改进, 从unsupervised变成semi-supervised的network.
  2. PS: 文章说的permutation invariant指的是排列不变性(个人认为因为是使用神经网络进行拟合的所以只是要求有这个功能, 并不要求有这个完全严格的输出值的限制), 应该是对pixel进行冲排列之后不会造成影响(所以因此不适合对conv进行这样的约束, 因为CNN考虑的是local信息)
  3. 提到了半监督学习中很多工作都进行了supervised和unsupervised的结合, 其中unsupervised任务包括了进行图像的重建或者能够将物体正确的分类.
  4. 提到了确实有同时使用无监督学习和监督学习的工作, 比如用无监督学习去辅助监督学习的工作(使用无监督学习进行预训练, 然后用监督学习进行fine-tune), 但是作者认为通常无监督和监督学习的任务是不同的, 使用无监督学习得到的特征可能并不适合于监督学习.
  5. 本文工作是基于Ladder network的, 提到了Ladder Network的辅助任务是在model的每层去denoise representation. 模型结构是添加了encoder到decoder的skip connection的autoencoder, 同时Ladder Network的learning task和denoising autoencoder类似, 但是其对每层都添加noise, 认为skip connection有助于减轻模型高层次represent details的压力. (PS: 这也就是为什么加noise了, 不然很容易就直接学习到identity关系了). 本文的改进点是 传统的Ladder network只使用了unsupervised学习, 本文进行了和supervised learning的结合
  6. 本文贡献:
    • (1). 能够与监督学习方法兼容: unsupervised部分关注与supervised learning找到线索的相关细节(注意强调了”相关”). 并且可以添加到现有的前馈神经网络中, 文章后续表明了, 可以先用supervised learning得到一个state-of-art结果之后进一步使用非监督学习进行性能进一步提升
    • (2). Scalability resulting from local learning(???没翻译明白): 在top层的supervised target之外还有per level 的 local unsupervised target, 认为会使层次更加适合深度网络.
    • (3). Computational efficiency: 正常而言需要几倍的训练时间去添加解码器, 但是本文通过更有效的方法只需要一个很小的时间倍数

Method:

  1. 本文首先从因变量模型角度对ladderNet进行解释.
    • (1). 提到了因变量模型是很有吸引力的半监督模型, 因为其可以结合监督学习和非监督学习. 其中唯一的区别就是label是否可以被观察到. 尤其是层次隐变量模型, 岂能通过底层来描述细节, 高层次去关注那些更加抽象的具有不变性质的feature以更好地处理任务.
    • (2). 讲述了一下DAEs和DSS两个模型的区别: DAEs: (给定观察量, DAEs从加噪版的观察量学习去重构观察量)x加噪声->x’->重构得到x^, 任务目标使 x^与x的MSE loss最小. DSS: (重构隐变量, 给定观察量) DSS训练一个映射函数f(x)->z, 来获得因变量z, 然后对z进行归一化之后使用一个降噪函数g(z_norm)->z^, 得到一个重构量z^, 任务目标是使 z^与z的 (这里强调了一下一定要对z进行归一化, 否则g会学习到恒等函数)
    • (3). LadderNet结构: 首先进行x的输入, 然后加噪进行一个映射关系的到第一个隐变量z^1, 然后对z^1加噪得到z^1’….直到z^n’, 然后会之后利用这个一层层的去还原(降噪)z^i, 同时每个z^i(skip connection)和z^i+1’都会共同来去学习还原z^i. 每层都是有损失函数的贡献的(z^i与z^i’的MSEloss) z^i是用另外的一个不加噪编码的分支得到的
    • (4). 本文的有监督部分是在加噪过程的最后的那个hidden layer后面利用这个representation去学习针对分类的任务, 对此进行有监督的训练, 无监督的训练指的是laddernet本身(也就是target为 加噪过程的降噪结果得到的还原隐变量和不加噪过程得到的隐变量的MSE loss)

PS: 个人认为本文的思路主要就是为了解决AutoEncoder自监督学习以及有监督分类的任务目的的不同, 自监督的特征会强调保留原有特征信息去用来重构, 但是分类任务通常需要抽象的语义信息, 所以本文对加噪encoder部分的最后进行监听, 希望能够使得encode特征是有用的, 同时又能够利用自学习(非监督学习)很好的约束优化(因为这时候还没有好的初始化方法去使得直接训练网络达到一个很好的效果)

PS: 具体的分析有点看不懂, 只是大致理解思路而已, 有需要的时候回来看后面

Introduction:

DAs尝试着去重建noisy inputs. 会通过想原始clean input添加根据input image的特征得到的基于一个noise distribution中的随机变量 -> corrupted input.

    • (2). 本文使用的CAEs则就是之前提到的 -> 利用局部的特征. 整个处理的过程的数学公式都是类似的, 只不过是用卷积和激活函数代替一下. 同时为了减少参数自由度去使filter得到的是图片整体内一个相同类型的feature, 所以使用的bias是公共的, 而不是每个pixel一个. 激活函数选择的是 tanh. 使用 MSE 作为 Loss Function.
  2. Max-Pooling: 这里说max-pooling layer的作用通常是用来获取具有位移不变形的representation. 认为其有助于提高filter的选择性. 本文选择使用max-pooling的原因是希望feature detector能够变得更加的广泛被应用, 避免那些没有鲁棒性的琐碎的模式被学习(比如identity). 在reconstruction阶段, 系数的latent code 将会减少每个pixel进行decoding的filter的平均数量, 使得filter更加general.
  3. Stacked Convolutional Auto-Encoders (CAES): 将一些AEs进行堆叠得到一个深层次的结构就是CAES了.

Simple Summary for Learning Deconvolution Network for Semantic Segmentation

Posted on 2019-11-09

简单总结《Learning Deconvolution Network for Semantic Segmentation》

Introduction:

  1. 本文主要思想就是学习一个深的deconv network.
  2. 强调了传统方法会出现的一些问题: 对待极端大物体会出现属于同一个物体pixels的不一致lable(就是物体太大了, 不能完全辨认, 被network辨别成了多个不同的物体), 对待小的物体可能会被忽略掉. 以及物体的细节结构很容易被丢失或者平滑掉, 因为输入到deconv层的label map相对来说太粗粒度了, deconv过程也相对太简单. 强调了FCN在得到1616的label map之后直接就deconv到原图尺寸, 而没有用大规模数据集去训练一个deep deconv network, 也因此使得重建一个关于物体边界的highly non-linear structure 变得十分困难, 因此当前很多工作使用CRF去解决这个问题. **PS:总结起来就是 目标的细节结构容易缺失, 边缘信息保留不够好 认为原因就是 -> FCN得到的label map过于粗糙, deconv过程太过简单(所以本文基于这个观点进行了问题的解决)***
  3. 本文使用了不同的策略, 贡献如下:
    (1). 学习deep deconv network, 包括了deconvoluiton, unpooling以及rectified linear unit layers.
    (2). 训练的网络被应用在individual object proposals上去获取instance-wise segmentations, 与最终的final semantic segmentation相结合.
    (3). 达到了一个很好的效果

Method:

  1. architecture: 整个网络就是由 (1). convolution part (2). deconvolution part 两部分构成的. 强调conv part就是将原图像进行特征提取, deconv part就是将提取到的信息用来进行object segmentation的生成. 最终的输出是和原图像相同尺度的probability map.
  2. conv part使用 VGG 16, deconv part是 conv part的镜像结构, 但是操作就取而代之的使用deconv, unpooling, rectification layers等.
  3. unpooling:
    • (1). 关于pooling, 强调了虽然是为了过滤掉没有必要的多余的”噪音”从而对classification任务引入了鲁棒性所以达到了很好的效果, 但是pooling过程中, 空间语义是丢失了的, 可能会对精确地定位产生了大的影响. 所以本文提出了在deconv part 中的 unpooling layers, 其记录了当时pool的过程中选择了留下了各个感受野中的哪个元素的坐标, 然后在unpooling过程中在那个记录的位置进行信息的填充.
  4. deconv:
    • PS: 本文所指的deconv部分, 并不是传统意义上的反卷积结构, 而是将unpooling得到的系数的activations值通过类似convlution的方法去进行处理.
    • (1). 文章认为在deconv layers中学习到的filters是用作还原input object shape的. 因此进行了一系列的deconv layers去捕捉不通过level的shape details.
    • (2). 发现低层次的filtrs倾向于去捕捉一个物体的总体的形状, 在深层次的filter中倾向于encodeclass-specific fine-details -> 网络可以直接用class-specific shape information去进行semantic segmentation.
  5. 后面部分内容暂时无关… 略过…

Simple Summary for Stacked Hourglass Networks for Human Pose Estimation

Posted on 2019-11-06 Edited on 2019-11-07

简单总结《Stacked Hourglass Networks for Human Pose Estimation》

PS: 主要去了解那个encoder-decoder结构
PS: 我有充足的理由觉得AAAI2019的M2Det的思想来源和这篇文章是有关系的.

Introduction:

  1. 本文针对的任务是pose estimate, feature会在所有的scales上进行处理并且consolidated去更好的捕捉身体的各种空间关系. 本文展现了反复的进行bottom-up、top-down并使用intermediate supervision是提升网络性能的关键. 本文的网络被称之为 “stacked hourglass” network, 其基于连续的pooling和upsampling操作去生政最终的一系列predictions.
  2. 提到了传统的关于处理pose estimation系统需要能够处理模糊、新的姿态、光照等问题的处理方法是(1). 使用鲁棒的image features, (2). 进行sophisticated structured prediction. 前者是用来生成局部的解释, 后者则是用来进行全局的pose推断.
  3. 称之为hourglass是因为先进行pooling然后进行upsampling. stacked hourglass是因为作者在后面还会使用多个pooling&upsampling的结构进行堆叠.
  4. 强调了本作模型设计的新颖是对称结构, 认为相比于传统的深度学习模型, 传统的进行pose estimation的深度模型其在top-down处理的过程中比较light, 而在bottom-up的过程中比较heavy, 而hourglass module则是两者之间对称的
  5. 强调了在没有对hourglass module进行stack的时候, 可以将单个的module就是一个conv-deconv的encoder-decoder结构[42-45]. 提到了[42]是用conv-deconv结构进行semantic segmentation的完成. [44]用这个结构去进行objects reflectance map的预测. [43]提出了一个unifie framework去进行supervised, unsupervised, semi-supervised learning(通过添加一个reconstruciton loss). [46] 使用encoder-decoder结构而不是用skip connections去进行image generation. [47] 提出了一个denoising auto-encoder with special “modulated” skip connections 去进行unsupervised/semi-supervised feature learning.

Method:

  1. Hourglass Design:
    • (1). 设计hourglass的目的就是因为需要在每个scale捕捉information的需求. 认为local evidence对于识别比如脸和收的features十分重要, 但是一个很好的pose estimate需要对整个body各个部分的理解. 人的方向, 对肢体以及邻近关节的relationships被认为是在图像多尺度中最容易被识别的线索之一. 认为hourglass是一个简单的, 极小的设计, 其具有步骤所有特征并将它们放在一起去产生pixel-wise predictions的能力. (个人认为因此需要深层次网络)
    • (2). 认为network必须要有一些机制去有效的处理并加强不同scales之间的features. 有些方法使用不同的pipeline去独立的处理图片, 然后将这些features在后续的network中进行结合.(应该指的就是使用image pyramid) 但是本文选择使用使用单一的带有skip layers的pipeline去保留各个尺度中的控件信息.
    • (3). hourglass模块的流程是: 先使用conv以及max pool去将features downsample到一个很低的resolution. 每个max pool处都会开一个分支用更多的conv去进行进行处理. 达到最低的resolution之后, network开始进行upsampling, 并且跨尺度的进行feature combination(将downsample的对对应resolution为i的经过后续conv处理的结果和当前的feature进行upsample之后的进行dot add操作, 得到下一个feature map). PS: 选取的是nearest neighbor upsampling.
    • (4). 达到network output resolution之后, 两个1*1 conv会进行使用去产生最后的network prediction结果. 结果是heatmap, 预测关节点的存在.
  2. Layer Implementation:
    • 主要说明了一些影响实验结果的细节.
    • (1). 比如其同时期的作品表明: 使用11 conv去减少计算量, 使用小的conv去代替大的conv. 所以作者最后通过实验表明选择使用residual modules效果是更好的. 同时不会选择使用超过33大小的conv.
    • (2). 提及到input resolution大小是256256, 但是这样需要很大的GPU的计算量, 所以最高的resolution of hourglass是6464(final output resolution). 整个network首先使用7*7步长为2的conv进行处理, 之后跟上一个residual module, 然后一个max pooling使得resolution从256下降到64. 之后在进行两个residual模块的处理, 之后就是hourglass module.
  3. Stacked Hourglass with Intermediate Supervision:
    • (1). 认为堆叠的hourglass结构可以提供initial estimates和整张图片的一个reevaluation.
    • (2). 提到了这个方式的关键部分是对中间的heatmap的prediction进行一个loss的约束. 每个hourglass module结束之后都会进行一个prediction. 认为subsequent hourglass modules使得高层次的features能够重新的被处理, 并且进一步验证以及重新评估更高层次空间关系
    • (3). 关于single hourglass module的使用, 作者认为我们可能无法确定在pipeline中的那个地方进行initial set of prediction的生成. 因为高层次的feature通常存在于low resoluiton, 是在upsampling操作前的非常靠后的位置. 如果supervision被提供在network之后, 那么就没有机会去将这些features在一个更大的全局上下文中被相互的重新验证. 同时如果我们想要network产生一个好的预测, 这些预测不能只在local scale上的验证. general context, 与其他关节预测的relationship以及对整张图片的理解都是至关重要的.
    • (4). 同时认为在pipeline的早期(pooling之前)进行supervision是可行的, 但是这时候的features仅仅是经过几个conv module的具有相对小的局部感受野的结果, 忽略了关键的全局线索.
    • PS: (3). 中对放在hourglass后进行验证的描述没有看明白… 是觉得网络不够深而导致了深层次的语义信息提取的不够, 并且没有足够的与浅层信息进行交织??? (4). 中放在前面是几个意思? 后面放的就是摆设了?
    • (5). 认为重复的bottom-up, top-down inference with stacked hourglasses可以减轻这些问题. 局部和全局的线索可以在每个hourglass module中进行整合. -> 个人认为就是在模块后面还会做一个浅层次和深层次语义信息的融合.
    • (6). 认为这么做是很重要的, 因为spatial location在低层次是很好的保留的, 在高层次中语义信息又是丰富的, 所以为了定位的准确性, 应该将空间位置信息去进行保留. 但是对于pose estimation的问题, 有需要进行feature关联之间的学习
    • (7). 最后的监督还是加在了hourglass module之后, 只不过通过一个shortcut 去引入浅层次的精准的位置信息, 而且能够进一步的去加深高层relationship的学习两个目的. (shortcut+heatmap用conv再处理+hourglass module output再处理)
      PS: 说实话只是大概的去了解了怎么做, 具体的思想和做法还是和pose estimation领域相关的, 所以理解的并不是很深刻. 文章并没有怎么详细说明AutoEncoder相关的内容…

问题:

  1. 个人觉得基于point的anchor-free模型为什么使用hourglass作为backbone效果好的原因之一就是因为本文在introduction中说明的, 个人认为point heatmap prediction相对于anchor-based来说其对feature map特征的细粒度要求更高, 也就是introduction中的观点: 传统的进行pose estimation的深度模型其在top-down处理的过程中比较light, 而在bottom-up的过程中比较heavy.
  2. PS: 从直观上来看个人认为decoder-encoder结构具有的功能是去除噪声(这么说也不是很恰当因为其还多了一个shortcut分支), 这种stacked的做法就是反复的将feature map的噪声一步步的去除掉了, 但是文章理解的角度是反复的去将深层次和浅层次的信息进行融合交织, 在更大的context中去进行prediction的预测.

Simple Summary for FEATURE INTERTWINER FOR OBJECT DETECTION

Posted on 2019-11-06

简单总结《FEATURE INTERTWINER FOR OBJECT DETECTION》

PS: 本文是关于利用less reliable source的data set(本文是小物体数据部分)去辅助reliable的data set(本文是大物体数据部分)去进行训练的, 用了无监督的思想(主要要总结的也就是这个方面的思想)

Introduction:

  1. 认为一个训练的很好的model应该能够在分类的时候每类别内都有一个一致的分数(应该指的是我们通常需要将物体正确所属类别的分数训练到1, 不正确的抑制到0, 所以作者认为这就代表着同一类别的物体最后要得到的分数是一致的, 因此需要高层次的语义信息的相似(但是实际上会有softmax进行处理, 所以我们并不需要要求分数相似, 只需要保证该层次的对应的语义信息被充分的提取出来就好了啊.)) -> 需要samples之间的高层次语义信息尽可能的相像. 提及到前人工作为了解决这个问题 都是通过重新设计loss或者新的regularization constraints.
  2. 本文从一个新的角度出发, 对每个类别假设有两组feature set: 一组是reliable information, 另外一组是less reliable information. -> 希望可靠信息来源数据能够在训练中指导less reliable set的特征的学习(类似于teacher-student mimick的思想)去使得less reliable information向feature space的更加”紧实”的质心移动. 认为这么做同样会给reliable information set带来好处, 因为同类别的samples变得更加接近, 因此是更加容易去进行分类的.
  3. 本文将这个思想用来进行object detection, 作者认为小物体因为很多细节的丢失所以看做是less reliable set, high resolution的obj被看做是reliable set. 本文的interwiner是设计去减小两个sets之间的divergence的. 认为从可靠数据集去进行有效feature representation的生成是十分重要的, 本文后面引入了optimal transport(OT) theory到网络框架之中. Less Reliable Set中的Samples则在OT metric的辅助之下更好的对齐.
  4. 强调了interwiner是可插拔的, 且效果好的.
  5. 强调了传统的做法就是使用交叉熵, l1/l2 loss, loss项的正则化约束去解决feature representation问题, 来保证最小的intra-class variation以及最大的inter-class variation. 认为这些工作的目标和本文的都一致, 都是想要学习到特征空间中每个类别的更加compact的representation.
  6. 仍然也强调了小物体因为前向信息丢失的问题导致性能的退化, 作者做了一个可视化, 在使用intertwiner和不是用的时候进行对比, 图中效果确实是intertwiner能够使得同类别的representation更加紧凑.
  7. 强调两个重要的改进就是: (1). 使用了buffer去存储class-dependent historical representative. -> 提到了是因为在训练的一个mini-batch中可能并不会对同一个类别有大量的sample, 所以需要在训练的过程中动态的进行更新. (2). optimal transport(OT) divergence的引入, 作为一个复杂的正则化???. OT metric将两个分布的比较从高维度的特征空间映射到低纬度空间, 所以可以消除掉敏感性去更好的进行两个分布相似性的评估(相当于去噪?). 认为OT可以使低可靠性数据集与可靠数据集更好的对准.
  8. 使用feature intertwiner方法的网络作者称之为 InterNet. 代码
  9. Related work中关于小物体目标检测的描述: 提到了传统是有两种做法的 (1). 加强context信息的利用, 认为RF的增大是有利于补偿小物体损失的信息的 (2). 多尺度, 认为多尺度是一种将物体不同尺度进行normalization的方法. 提到了本文是一种另外的解决小物体检测的思路, 利用可靠大物体的帮助(其实GAN那篇文章也是利用了这个思想, 但是是从大物体和小物体辨别难度、gap方面入手的)
  10. Related work中关于loss function设计: 强调了传统的交叉熵是不具备限制类别间的能力的, 所以很多工作都致力于添加类别间的regularization.

Method:

  1. Feature Intertwiners for Object Detection:
    • (1). 使用Faster RCNN作为pipeline.
    • (2). 首先, 网络仍然根据feature map的空间大小分成一些level, 在每个level l, 会将region proposals 分成两个categories: 1). large-region set(reliable): 比RoI-pooling layer输出的size要大region, 2). small-region set(less reliable).
    • (3). 之后feature map P_l 将会fed到RoI layer, 然后传给一个make-up layer. make-up layer的作用是: fuel back在RoI中丢失的信息, 同时补偿small resolution obj instance的必要details. -> 认为经过这个层次的refined high-level semantics对各种因素(pose, lighting, appearance等)会更加的鲁棒, 消除了variation. 强调了这个make-up unit是通过interwiner unit进行学习和优化的, 其收到了大物体set的feature的帮助
    • (4). feature interwiner是衡量两个set之间数据分布的divergence的. 对于可靠集合, input直接就是RoI layer的output, 也就是高层次或者高resolution的samples. 对于less reliable set, input则是make-up layer的输出. 两个input都会fed到一个critic module中去提取两个集合进一步的representation, 并且提供给intetwiner evidence. critic包含了两个conv, 将feature transfer到一个更大的channel size但是降低了spatial size到1(就是降维成了一个一维向量). 然后使用一个简单的l2 loss去比较两个set之间的不同.
    • (5). 最后的loss就是各个scale上标准detection loss和intertwiner loss的综合.
    • (6). 仍然面临着两个问题: 1). 通常在一个mini-batch中并不会出现同一个类别的instance在两个set中分别出现的情况(用之前提到的buffer解决) 2). 如何去进行这两个set的划分
    • PS: 其实后面思想可以不用看了, 主要目的就是减小大物体(可靠物体)和小物体(不可靠物体)深层信息表达上的差异性来减少最后错误分类的概率. 靠的就是用loss来减少差异
  2. Class Buffer:
    • (1). 存的是这个类别的large instance的平均feature representation. (并不是所有, 那其实感觉挺依赖于前面的几个??)
    • (2). 可以采用不同形式的方法去处理feature representation, 但是实验中表明是mean效果最好
  3. interwiner的一些问题的说明:
    • (1). 谈及使用大物体去对小物体进行学习的指导, 就是teacher-student mimic的思想, 希望小物体和大物体在visual appearance上的variance尽可能的小, 而不受到resolution的影响.
    • (2). 提到了buffer是不应该参与到training更新的过程中的
    • (3). buffer statistics是从各个尺度上获得的, 强调large和small是相对的, 但是认为level-agnostic buffer通常来说总会接受到strictly large instance的语义特征 -> 认为这就是为什么实验会对所有尺度都有提升的原因.
  4. Choosing best feature map for large objects using optimal transport:
    • 选择那一层的特征图作为input source的feature map是十分重要的(就是选teacher的input).
    • (a). 选择当前层的feature map: 认为是不合适的, 因为当层被用来在RPN内训练针对于识别小物体的, 用这一层作为input可能会导致一些small instances的noisy details被引入
    • (b). 选择更深(小)层次的feature map: 认为这相对于来说是对大物体来说更加合适的尺度, 同时深层次具有更小的resolution和更高的予以层次. 但是对于这些层次中的大物体, 可能在其所在层次上是小物体, 因此其在这些高层次的feature map上可能没有携带足够的信息, 应该在RoI操作中被upsample, 去更新当前尺度的buffer. 同时试验表明了在高层次但是在高层次中是小物体的现象是十分常见的, 所以这么直接做是不合适的.
    • (c). 先把深层次的feature map upsample到当前层次的resolution上, 然后使用RoI-pooling去处理这个放大后的feature map. 可以选择的几种设置: 1). 双线性插值 2). 使用神经网络(???deconv吗) 后者试验表明效果要远好于前者
    • (d). 使用本文提到的OT: 提到了本文的最优的将一个distribution move到另外一个distribution是符合optimal transport(OT) domain的. 本文将OT unit在P_l与P_m|l(就是放大的深层次feature map)进行之间插入, 其是作为RoI-pooling操作之前的inputs的. PS: 没太看明白… 以后有需要仔细看…

Simple Summary for SNIPER: Efficient Multi-Scale Training

Posted on 2019-11-06

简单总结《SNIPER: Efficient Multi-Scale Training》

Introduction:

  1. 本文提出的SNIPER是进行instance level visual recognition任务多尺度的高效训练的算法. 并不去进行image pyramid所有像素的处理, SNIPER只处理GT instance在合适的scale上周围的context的信息. 说bg sampling 的 context-regions都是通过一个短周期训练的RPN网络提取出来的, 因此每个image在训练中生成的chips的数目都是根据场景复杂度自适应的改变的. SNIPER在一个单尺度(8001333)上仅仅需要处理30%左右的pixels即可, 而且其可以对极端大的resolution的图像进行处理(其实太大了导致物体超过了512512肯定是不行的), 提到了SNIPER针对重新采样的low resolution chips(512*512)进行操作 -> 因此可以在GPU上采取一个很大的batch(说因此就可以直接享受到batch-normalization的好处, 而不用跨GPU的去进行synchronizing batch-normalization). 作者认为, SNIPER将原图像划分为一个个的小的chips使得instance level recognition任务(主要指detection)更加倾向于image classification. 同时认为传统普遍接受的观点: 在high resolution图片上进行visual recognition十分重要可能并不是正确的(…其实你不也只是从大的图片上截取下来的吗…).
  2. 又是从人眼角度开始描述的, 同样认为人们只会关注一小部分的图片而周围的区域将会是模糊的, 所以是难以同时处理多尺度的. 通常是仅仅处理一小部分的image, 然后自适应的忽略掉图像剩余部分的视觉信息, 而主流的detector为了robust的scale representation而多尺度的去进行训练, 并且其会对所有的pixel进行训练. 同时认为多尺度会因为放大且全处理导致时间的大量增长, 在很多情形下是不可行的.
  3. 提到了之前自己的SNIP, 说在训练的过程中忽略掉特定scale下extreme resolution objects的梯度是有利于训练的. -> 也就是在特定尺度上关注特定大小的物体 -> 一个符合直觉的想法就是, 我们是不是可以不用将整张图片放大, 而只是sample在小物体周围的一个小的RoI chip就可以.同时从另一个角度说明这个选择就是, 当一个图像已经在一个high resolution了, 并且其中有很大的obj了, 那么upsample这些大的obj可能并不会带来好处(个人认为反而会因为感受野并不够大而错误判断一些事情)
  4. 作者接下来又说, 确实放弃掉image中很大一部分然后在upsample有助于节省计算量, 但是一个小的chip可能会因为缺少contexual信息去进行识别. 很大一部分的背景信息可能也会被忽略掉 -> trade-off between computation, context, negative mining. 也就是本文提出的训练算法(SNIPER), 其根据image content去自适应的从image pyramid不同尺度上去sample一些chips出来.
  5. 会根据GT instances sample一些positive samples出来, 根据RPN网络的proposal sample一些negative samples出来.
  6. 主要贡献: (1). 保证和SNIP相似性能的同时 可以减少3倍的处理pixel的数量. (2). 因为SNIP patch是比较小的, 所以可以享受到大的batch-normalization带来的好处(单卡20, 8卡160的batch-size, 即使使用的是resNet-101这种大backbone)
  7. 提到了关于R-CNN与Fast-RCNN的一个对比, (本文主要是通过截取固定大小避免了很多没有意义的物体的放大与缩小这一角度出发, 但是一篇知乎上的解读认为可以看做是对RCNN具有尺度不变性和F-RCNN的没有的分析, 虽然文章没有这么表述, 个人也并不认为本文是从这个角度出发的)(deformable conv2从另外一个角度进行这个事情的解释). 提到了F-RCNN会对所有image中的proposal进行upsample与downsample. 和RCNN不同, RCNN每个proposal都会因为SS算法而固定到224*224的大小, large obj与small obj都没有进行upsample与downsample. 而是只处理当前尺度下包含合适resolution的物体的区域, 认为这是比较合适的(文章中的角度主要是指没有将大物体放大, 小物体缩小这种不恰当的处理, 但是实际上这么做在de-conv2中解释是有利于尺度不变性), 但是Fast-RCNN与Faster-RCNN中已经明确分析了, 因为SS没有共享feature map以及SS算法是基于CPU进行的所以速度很慢. 本文认为提出的SNIPER算法享受了两者的好处, (1). 生成尺度相关的固定大小的chips, 并且使chips中包含尽可能多的合适scale的物体 (2). 在一个chips内使用Fast RCNN的做法, 去进行chip中多proposal的处理. -> 以上两点带来了好处就是, 不需要在包含大物体的地方进行upsample, 同时不用进行简单bg region的反复处理了. 相比于Fast-RCNN显著要快.
  8. 在SNIP的基础上, 对SNIP的 480, 800以及多个从1400 crop出1000大小的image pyramid进行处理, 去sample 512大小的patch. 实验发现, 在比较大(3*)的image上, 计算量可能仅仅是原先的1/10. 但是遇到的一个问题是, 当仅仅使用GT instance去生成patch的时候, 因为仅仅处理图像的部分内容, 导致训练中很少会对背景信息进行学习处理, 所以导致了假正率的很高, 因此后来还是要对bg进行negative patch的sample(sample可能包含false positive的region)(使用一个轻量级短周期训练的RPN去生成这些区域).

Method:

本文包含了两个模块: (1). positive/negative chip生成 (2). chip生成后的label assignment

  1. Chip generation:
    • (1). 首先SNIPER会在多个尺度{s1, s2 …}上进行chip的生成, 每个尺度上, 首先iamge会resize到 $W_i H_i$ 上, 然后 K K 大小的chips将会没 d pixels(本文是d=32)的进行生成(其实就相当于一个大的anchor… )
  2. Positive Chip Selection:
    • (1). 在每个scale上, 指定了一定的area range, 决定了每个scale上参加训练的GT boxes/proposals. 这些GT bbox构成了一个集合, 然后我们贪心的去选择能够将cover valid GT box最多数量的patch, 只有当一个GT box被完全的覆盖在chip中的时候才被称之为covered. 一个scale中的每个picture的所有的positive chip将会被combine 在一起.
    • (2). 上面的贪心保证了每个GT box都至少被一个patch所包含, 但是可能会存在一个GT bbox被同一个scale上的多个chip所包含, 也可能存在在不同的scales上被不同的chips所包含.
    • (3). 没有被完全被chip包含的GT box是被crop的, 但是cropped GT box仍然会保留在chip中, 并用来进行label assignment(???什么意思???)
    • (4). 至此, 所有的GT box都会在合适的scale上被covered. 同时提到了 因为crop-size是远小于图片resolution的, 因此SNIPER并没有在high resolution上对很多的背景信息进行处理. 这在处理high resolution image的时候大幅度的减少了计算量与内存需求.
  3. Negative Chip Selection:
    • (1). 提到尽管positive chips覆盖了所有的positive instances, 但是很大一部分比例的bg并没有被覆盖到, 因此导致了在训练的过程中缺少背景的训练, 导致了在inference的时候false positive的上升. 提到当前点的object detection算法中, 通常multi-scale training是经常被使用的, 在所有的scale上图片的每个pixels都会被训练, 这可以有效的减少false positive, 但是却带来了大量的计算损耗, 并不像这么做(因为这么做就发不了paper了 2333).
    • (2). 作者假设很大一部分比例的bg是很容易被区分的(算是常识… imbalance问题), 所以作者认为可以避免去使用这些bg region去进行训练.
    • (3). 为了去找到这些区域, 作者认为最简单的方法就是使用object proposals去找出obj可能存在的区域, 剩下的区域就是被认为很简单就可以区分的区域, 这些区域就是我们应该放弃的.
    • (4). 实现上面的方法, 首先用一些epoch去先训练一个RPN, 不适用negative chips去进行训练, 因为这个网络的目的就是帮助我们粗略的找到可能存在物体的regions(文章说的是可能包含false positive的区域, 其实是一个意思), 这个RPN不需要非常精准.
    • (5). 对negative chip的选择时, 首先不考虑那些很简单就能区分的区域, 然后排除哪些与positive chips cover的proposal. 之后对于每一个scale, 我们类似选择贪心的去进行选择negative chip, 每个chip都只要要包含M个proposals, 这个时候就得到了最后的negative chips set
  4. Label Assignment:
    PS: 需要注意的是, 进入训练的是每个patch, 而不是original input image pyramid.
    (1). 强调了本文是end2end训练的, 每个chips都会类似于Faster-RCNN去进行训练.
    (2). 关于RPN的训练, 不进行尺度的区分(所以当一个大物体被一个chip crop的时候所剩下的小的region仍然是在这个阶段valid的), 但是后续进行proposal处理的时候只有落在当前尺度下的GT box才会被用来训练, 其他的将会被忽略掉. 剩下的设定和Fast-RCNN相似.
  5. Benefits: 大部分点都是之前提到过的, 另外其提及到本作可以一定程度的表明, 图片中有相当一部分的context对检测是没有什么作用的, 但是也有可能是因为深度网络还不够深, 其感受野(经验感受野)不足以覆盖全图的所有pixels.
  6. Inference过程: 使用图像金字塔: (480, 512), (800, 1280), (1400, 2000)的image pyramid(括号中的数字描述的是范围信息), 然后谈到了训练和推断期间有效的区域是和SNIP相似的. combine detections 使用的是soft-NMS
  7. 关于为什么对那个筛选可能存在物体的区域的RPN只使用 positive sampling 进行训练 -> 作者进行实验发现, 当RPN没有进行negative sampling的时候 召回率是没有下降的(与mAP相反), 因为召回率并不考虑假正率, 所以可以认为只使用positive sampling进行训练的时候, 已经可以发现所有的positive regions. 个人认为, 我们仅仅使用positive sampling可以加快收敛可以大大减少这个RPN的训练时间.
